# Glossar

Sie könnten in der Rocket Pool Dokumentation auf einige unbekannte Konzepte stoßen. Dieser Abschnitt listet die gängigen Begriffe in der Dokumentation für einfachen Zugriff, Lernen und Plugin-/Theme-Entwicklung auf.

## APY - Annual Percent Yield (Jährliche prozentuale Rendite)

Der Betrag des Gewinns (Rendite), der über einen Zeitraum von einem Jahr berechnet wird, geteilt durch den anfänglichen Investitionsbetrag, ausgedrückt als Prozentsatz.

## Bonding

_Siehe Insurance._

## Contract, (Smart)

Ein Ethereum-Programm (Smart Contract), das zur Ausführung einer Reihe programmierter Funktionen verwendet wird. Rocket Pool Smart Contracts umfassen den Deposit Pool, die Minipool Validator-Bildung und geben verschiedene Token-Interaktionen auf der Ethereum Chain aus und verfolgen diese.

## Client

Ein Client ist eine Implementierung von Ethereum-Software, die alle Transaktionen in jedem Block verifiziert und das Netzwerk sicher und die Daten genau hält. _Siehe auch Execution und Consensus Client._

## Custodial

Zentralisierter Service, der den gesamten ETH-Staking-Prozess im Namen des Nutzers verwaltet und die "Verwahrung" über private Validator-Schlüssel und Withdrawal-Schlüssel behält. _Siehe auch Non-Custodial._

## DAO (Decentralized Autonomous Organization)

Eine Organisation, die durch Regeln repräsentiert wird, die als Computerprogramm kodiert sind, das transparent ist, von den Organisationsmitgliedern kontrolliert wird und nicht von einer zentralen Regierung beeinflusst wird.

## Deposit Pool

Die Menge an ETH, die zum Staking bereit ist. Der Deposit Pool wird von regulären Nutzern finanziert, wenn sie ETH gegen rETH tauschen. Dieses ETH ist der Betrag, der darauf wartet, mit Node-Betreibern in der Node-Betreiber-Warteschlange gepaart zu werden.

## ETH

Die grundlegende Kryptowährung, die vom Ethereum-Protokoll generiert und verwendet wird.

## Execution Client

Ethereum-Software-Clients, die die Execution Layer betreiben.

## Consensus Client

Ethereum-Software-Clients, die die Consensus Layer betreiben.

## eth2.0 (die Beaconchain)

Das Ethereum 2.0 Netzwerk und Update des Ethereum-Netzwerks, das im Dezember 2020 gestartet wurde und den Proof-of-Work eth1.0 Konsensmechanismus im September 2022 ersetzte. Veralteter Begriff - jetzt einfach als Beaconchain oder Consensus Layer bekannt.

## Insurance

Die Menge an RPL-Token, ausgedrückt als Prozentsatz des Wertes des vom Protokoll bereitgestellten ETH, die ein Node-Betreiber einzahlen muss, wenn er einen Minipool Validator startet. Die von einem Node-Betreiber gestakte RPL-Versicherung dient als Versicherung, um reguläre Nutzer zu entschädigen, falls der Node-Betreiber das Staking mit weniger als dem vom Protokoll bereitgestellten ETH in seinen Validatoren beendet (24 ETH im Fall eines 8 ETH Minipools, 16 ETH im Fall eines 16 ETH Minipools).

## Minipool (Validator)

Ein Validator, der über die Rocket Pool Software finanziert und initialisiert wurde. Er besteht aus ETH, das von regulären Nutzern über den Deposit Pool beigetragen wurde, und ETH, das von Node-Betreibern während ihres Registrierungsprozesses beigetragen wurde.

## Node

Im Ethereum-Netzwerk sind Nodes Geräte (Computer), die Ethereum-Client-Software ausführen.

## Node Commission

Der Betrag an zusätzlichem ETH, der einem Node-Betreiber zuerkannt wird. Dies ist effektiv die "Gebühr", die von normalen Stakern genommen wird; bei Rocket Pool wird diese Gebühr an die Node-Betreiber gegeben. Als Rocket Pool startete, lag sie zwischen 5 - 20% mit dem idealen Ziel von 10%, wobei die Provision jedes Minipools bei der Erstellung des Minipools lebenslang festgelegt wurde. Neue aktivierte Minipools haben jetzt eine festgelegte Provisionsrate von 14%.

## Node Operator

Eine Person, die das Ethereum-Netzwerk sichern möchte, indem sie eine Node (Computer) betreibt, die die Rocket Pool Software, einen Validator Client und optional auch einen Execution und Beaconchain Client ausführt. Node-Betreiber verdienen ETH APY-Belohnungen zu einem höheren Satz als Solo-Staker im Ethereum-Netzwerk.

## Node Software

Eine Reihe von Programmen, die es Node-Betreibern ermöglichen, mit dem Rocket Pool Protokoll zu interagieren.

## Node Wallet

Eine Ethereum Wallet, die von Rocket Pool beim Erstellen einer neuen Node generiert wird. Diese Wallet wird verwendet, um neue Minipools während ihrer Erstellung zu finanzieren und die Gasgebühren zu bezahlen, die für die Interaktion mit den Smart Contracts von Rocket Pool benötigt werden.

## Non-Custodial

Ein Service, der die Optimierung für die Validator-Einrichtung und -Verwaltung bereitstellt, aber keine privaten Validator-Schlüssel und Withdrawal-Schlüssel der Nutzer hält. Rocket Pool ist ein Non-Custodial Staking-Service.

## Oracle DAO (oDAO)

Eine DAO, die aus allen Oracle-Node-Betreibern besteht. _Siehe auch DAO._

## Oracle Node

Eine spezielle Node, die von vertrauenswürdigen Mitgliedern der Rocket Pool Community betrieben wird und Informationen von der Beacon Chain (wie Gesamt-Staking-Belohnungen und Validator-Exit-Status) zurück zur Execution Layer weiterleitet, damit die Rocket Pool Smart Contracts korrekt funktionieren können.

## Proof-of-Stake (PoS)

Der Konsensmechanismus, der in Ethereum verwendet wird, bei dem Nutzer über die Korrektheit neuer Blöcke abstimmen, indem sie das ETH in ihrer Wallet staken.

## Protocol DAO (pDAO)

Die DAO, die aus allen Inhabern von RPL-Token besteht. _Siehe auch DAO._

## Regular Node (Operator)

Der Begriff für "normale" Node-Betreiber, die nicht die zusätzlichen Verantwortlichkeiten von Oracle Nodes haben.

## Regular Stakers

Ein Privat-(Regular-)Nutzer, der Zinsen auf seine ETH-Münze verdienen möchte, ohne sie vom zugrunde liegenden Wert von ETH zu entkoppeln. Ein Regular Staker interagiert mit der Rocket Pool Plattform, indem er ETH-Münzen gegen rETH-Token tauscht. Ihr ETH wird in den Staking Pool eingegeben, wo es von Node-Betreibern erworben wird, wenn sie neue Minipools erstellen.

## rETH (Rocket Pool Staking Deposit Token)

Ein Token, der die Einzahlung eines regulären Nutzers in den Staking Pool repräsentiert. Nutzer erhalten einen gleichwertigen Wert an rETH-Token für jede ETH-Münze, die sie einzahlen. Wenn durch Staking im Ethereum-Netzwerk Zinsen verdient werden, steigt der Wert des rETH-Tokens in Bezug auf ETH. Dieser Token muss nicht im Netzwerk gesperrt werden und kann nach Belieben des Nutzers gehandelt, verkauft oder gehalten werden.

## RPL (Rocket Pool Token)

**RPL — Rocket Pool Protocol Token**

RPL ist der primäre Protokoll-Token, der für die Governance des Protokolls verwendet wird und auch auf einer Rocket Pool Node gestakt werden kann.

Aus Sicht eines Node-Betreibers:

- Ab Saturn 1 erhalten wahlberechtigte gestakte RPL einen Anteil am gesamten Protokollumsatz
- Die [Mehrheit](https://rpips.rocketpool.net/RPIPs/RPIP-25) der RPL-Inflation wird als Belohnung an Node-Betreiber basierend auf gestaktem RPL gegeben; diese Belohnungen sollen mit Saturn 2 auslaufen
- Für Saturn 0 Minipools (erstellt nach dem 28.10.2024): Gestaktes RPL bietet bis zu 4% zusätzliche Bonusprovision; die Bonusprovision soll einige Monate nach dem Start von Saturn 1 auslaufen.
- Gestaktes RPL bietet Stimmrecht für die Governance

Aus Protokollsicht:

- Eine [Minderheit](https://rpips.rocketpool.net/RPIPs/RPIP-25) der RPL-Inflation stellt Finanzierung für die pDAO bereit
- Eine [Minderheit](https://rpips.rocketpool.net/RPIPs/RPIP-25) der RPL-Inflation stellt Finanzierung für die oDAO bereit
- Es kann als sekundäre Sicherheit dienen (z.B. gegen Slashing)

## Stake

Ein Betrag an Münze oder Token, der als Sicherheit hinterlegt wird, um eine Arbeit auszuführen. Eine Person kann ETH entweder als regulärer Nutzer durch Tausch von ETH gegen rETH oder als Node-Betreiber durch Einzahlung von ETH zur Verwendung als Teil des Beaconchain PoS-Konsensprozesses staken. Eine Person, die ihr ETH staked (z.B. ein Staker), wird mit einem APY oder einer Rendite auf ihr eingezahltes ETH belohnt.

## Staking Pool

Ein Staking Pool ermöglicht es mehreren ETH-Münzinhabern, ihre Token zu kombinieren, um gemeinsam den erforderlichen Betrag zum Betrieb eines Ethereum Validators zu bilden. Rocket Pool bildet, verfolgt, weist zu und versichert diese kollektiven Pools mit einer Non-Custodial dezentralen Methode.

## Validator

_Siehe Minipool._
