# Migration der Smartnode von früheren Beta-Tests

Wenn Sie während unserer früheren Beta-Tests auf Medalla oder Pyrmont ein Rocket-Pool-Node-Betreiber waren, danken wir Ihnen für Ihre Hilfe bei der Verbesserung des Protokolls und heißen Sie willkommen zurück!
Dieser kurze Leitfaden führt Sie durch den Prozess der Bereinigung Ihres eigenen Beta-Builds und der Einrichtung des neuesten Builds entweder für das Hoodi-Testnetz oder für echtes Staking im Mainnet.

<div className="p-3">
  <Tabs>
    <Tab label="Hoodi Network">
      <p className="rspress-directive warning">
        <p className="rspress-directive-title">HINWEIS</p>
        Sie müssen ein neues Node-Wallet erstellen, um mit dem Hoodi-Testnetzwerk zu interagieren.
        Die Wiederverwendung eines Wallets aus einem früheren Beta-Build kann zu Fehlern beim Erstellen neuer Validatoren führen!
      </p>

      Es gibt drei Hauptschritte beim Upgrade von einem Beta-Build zum Hoodi-Testnetzwerk.

      Migrieren Sie zunächst alle Hoodi-ETH, die Sie haben, zu einem separaten Wallet.
      Sie können sie dann nach der Neuinstallation von Rocket Pool zurück zum neuen Node-Wallet migrieren, sodass Sie sich nicht so sehr auf einen Faucet für Hoodi-ETH verlassen müssen.

      Der einfachste Weg dazu ist über Metamask - siehe die [rETH-Staking-Anleitung](../liquid-staking/overview) für eine Anleitung zum Einrichten eines neuen Wallets.
      Sie können dies mit folgendem Befehl tun:

      ```shell
      rocketpool node send XXX eth 0xabcd1234...
      ```

      wobei `XXX` die Menge an Hoodi-ETH ist, die gesendet werden soll, und `0xabcd1234...` die Adresse des neuen Wallets ist, das Sie erstellt haben.
      **Beachten Sie, dass Sie ein wenig ETH zurücklassen müssen, um für das in der Transaktion verwendete Gas zu bezahlen!**

      Entfernen Sie als Nächstes alle alten Docker-Images und Chain-Daten mit folgendem Befehl:

      ```shell
      rocketpool service terminate
      ```

      Sobald dies erledigt ist, entfernen Sie den alten Rocket-Pool-Daten- und Einstellungsordner.
      **Dies löscht Ihr altes Wallet und Ihre alten Validatoren. Stellen Sie daher sicher, dass Sie diese nicht mehr für irgendetwas speichern müssen!**

      ```shell
      sudo rm -rf ~/.rocketpool
      ```

      Sobald dies erledigt ist, wird Ihr altes Beta-Setup bereinigt.
      Sie können der [Rocket-Pool-Installationsanleitung](../node-staking/eth-clients) folgen, um eine Anleitung zur Installation und Konfiguration von Rocket Pool für das Hoodi-Testnetz zu erhalten.
