# Üben mit dem Testnetzwerk

Ein Ethereum-**Testnetzwerk** (`testnet` abgekürzt) ist eine Ethereum-Blockchain, funktional identisch mit der Standard-Ethereum-Blockchain, aber sie soll zum Testen von Smart Contracts verwendet werden, bevor diese in der "echten" Hauptblockchain (genannt `mainnet`) live gehen.
Testnets laufen parallel zum Mainnet - tatsächlich laufen derzeit mehrere davon!

Sie können [hier](https://support.mycrypto.com/how-to/getting-started/where-to-get-testnet-ether) mehr über einige der Execution-Layer-Testnets lesen.
Rocket Pool verwendet **Hoodi** für sein Execution-Layer-Testnetz.
Sie können hier einen Block-Explorer dafür verwenden, wenn Sie einen Blick darauf werfen möchten: [https://hoodi.etherscan.io/](https://hoodi.etherscan.io/)

Das gleiche Konzept gilt auch für die Consensus Layer.
Rocket Pool verwendet **Hoodi** für sein Consensus-Layer-Testnetz.
Sie finden hier einen Block-Explorer dafür: [https://hoodi.beaconcha.in/](https://hoodi.beaconcha.in/)

Da die Netzwerke zum Testen gedacht sind, **wird ETH darauf kostenlos verteilt, um Menschen beim Testen zu helfen**.
Dieses ETH **kann nicht** zum Mainnet übertragen werden; es muss auf Hoodi bleiben und hat keinen realen Wert, Sie können es also als **Monopoly-Geld** betrachten.

::: danger
**Verwenden Sie NICHT Ihr echtes ETH vom Mainnet, wenn Sie das Testnetzwerk von Rocket Pool ausprobieren!**
Sie werden **Ihre Gelder für immer verlieren**, wenn Sie dies tun!
Sie sollten nur Test-ETH verwenden, das Sie auf dem Hoodi-Netzwerk von einem Faucet erhalten haben (unten beschrieben)!
:::

## Konfiguration des Testnets

[Die Standardkonfiguration von Rocket Pool mit Docker](../install-modes) übernimmt automatisch alle Einstellungen für die Verwendung eines Testnets.
Es gibt nichts Besonderes, was Sie tun müssen.

Wenn Sie [ein anderes Setup](../install-modes) mit einem externen Execution- oder Consensus-Client verwenden oder Ihre Clients [nativ ohne Docker](../install-modes) ausführen, müssen Sie dies manuell einrichten:

- Für **Geth**: fügen Sie `--hoodi` zum Startbefehl hinzu.
- Für **Lighthouse**: fügen Sie `--network hoodi` zum Startbefehl hinzu.
- Für **Nimbus**: fügen Sie `--network=hoodi` zum Startbefehl hinzu.
- Für **Prysm**: fügen Sie `--hoodi` zum Startbefehl hinzu.
- Für **Teku**: fügen Sie `--network=hoodi` zum Startbefehl hinzu.
- Für **Lodestar**: stellen Sie dem Startbefehl `LODESTAR_NETWORK=hoodi` voran

Im nächsten Abschnitt erfahren Sie, wie Sie Test-ETH auf Hoodi erhalten, das Sie zum Testen von Rocket Pool benötigen.

## Test-ETH auf Hoodi erhalten

Um das Rocket Pool-Testnetzwerk zu nutzen, benötigen Sie ETH auf Hoodi.
Glücklicherweise können Sie dies kostenlos über einen **Faucet** erhalten - einen automatisierten Service, der Ihnen Hoodi-ETH zum Testen bereitstellt.

::: warning HINWEIS
Sie müssen eine Ethereum-Adresse angeben, um Test-ETH zu erhalten.

Wenn Sie nur versuchen, Ihr ETH normal zu staken und rETH zu erhalten, lesen Sie bitte die [Staking-Anleitung](../../liquid-staking/overview), bevor Sie Hoodi-ETH erhalten.
Sie werden aufgefordert, zum entsprechenden Zeitpunkt hierher zurückzukehren.

Wenn Sie planen, einen Node zu betreiben, gehen Sie bitte die [Node-Operator-Anleitung](../responsibilities) durch, um zuerst einen Node einzurichten und ein neues Wallet zu erstellen.
Sie werden aufgefordert, zum entsprechenden Zeitpunkt hierher zurückzukehren.
:::

Rocket Pool bietet einen Faucet auf seinem [Discord-Server](https://discord.gg/rocketpool).
Treten Sie dem Server bei und gehen Sie zum `#support`-Channel.

Hier können Sie nach Testnetz-ETH fragen.
Sie müssen Ihre Node-Adresse angeben.

::: warning HINWEIS
Der Testnetz-Faucet-Bot ist aufgrund früheren Missbrauchs nicht mehr direkt für die Öffentlichkeit zugänglich.
Er steht nur dem Entwicklungsteam und prominenten Mitgliedern der Community zur Verfügung.
Um Testnetz-ETH zu erhalten, führen Sie die unten stehenden Befehle aus und geben Sie Ihre Absichten im Channel an.
Seien Sie geduldig und jemand wird für Sie auf den Faucet zugreifen.
:::

Wenn Sie nicht planen, einen Node zu registrieren, und nur ETH zum Testen des Stakings möchten, um rETH zu erhalten, geben Sie den folgenden Befehl im Channel ein:

```
!hoodiEth <ihre hoodi adresse>
```

ohne die spitzen Klammern.
Wenn Ihr Wallet noch kein ETH hat, sendet der Bot 1 ETH zum Testen an Ihr Wallet.

Wenn Sie _planen_, einen Node zu betreiben, gehen Sie wie folgt vor:

1. Führen Sie zunächst den obigen `!hoodiEth`-Befehl aus, um genügend ETH zu erhalten, um Ihren Node zu registrieren
2. Registrieren Sie Ihren Node (siehe die Seite [Vorbereitung Ihres Nodes](../prepare-node) für Anweisungen)
3. Führen Sie den folgenden Befehl aus, um 8 Hoodi-ETH zu erhalten, damit Sie einen Validator erstellen können:

```
!hoodiOperator <ihre hoodi adresse>
```

Um `!hoodiOperator` zu verwenden, _muss_ die von Ihnen angegebene Adresse ein registrierter Rocket Pool-Node sein.
Daher müssen Sie diese 3 Schritte in dieser Reihenfolge befolgen.

## Test-RPL auf Hoodi erhalten

Wenn Sie den Node-Betrieb testen, benötigen Sie zusätzlich zu Test-ETH auch Test-RPL.

Rocket Pool bietet einen Faucet auf seinem [Discord-Server](https://discord.gg/rocketpool).
Treten Sie dem Server bei und gehen Sie zum `#support`-Channel.

Hier können Sie nach Testnetz-RPL fragen.
Sie müssen Ihre Node-Adresse angeben.

::: warning HINWEIS
Der Testnetz-Faucet-Bot ist aufgrund früheren Missbrauchs nicht mehr direkt für die Öffentlichkeit zugänglich.
Er steht nur dem Entwicklungsteam und prominenten Mitgliedern der Community zur Verfügung.
Um Testnetz-RPL zu erhalten, führen Sie die unten stehenden Befehle aus und geben Sie Ihre Absichten im Channel an.
Seien Sie geduldig und jemand wird für Sie auf den Faucet zugreifen.
:::

```
!hoodiRpl <ihre hoodi node adresse>
```

Dies stellt Ihnen 600 des **Legacy (v1) RPL**-Tokens zur Verfügung, das analog zum ursprünglichen RPL-Token im Mainnet ist.
Rocket Pool verwendet einen **neuen (v2) RPL**-Token, der die Inflation zur Belohnung von Node-Operatoren unterstützt.

Um zu erfahren, wie Sie Legacy-RPL gegen neues RPL tauschen, folgen Sie den Schritten in der Anleitung [Einen Minipool (Validator) erstellen](../create-validator).
