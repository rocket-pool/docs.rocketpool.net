# Vorbereitung Ihres Nodes für den Betrieb

Wenn Sie hier sind, haben Sie erfolgreich die Smartnode-Dienste gestartet, ein Wallet erstellt und die Synchronisierung sowohl der Execution- als auch der Consensus-Chains auf Ihren jeweiligen Clients abgeschlossen.
Wenn ja, dann sind Sie bereit, Ihren Node im Rocket Pool-Netzwerk zu registrieren und einen Minipool mit einem Validator zu erstellen!
Falls nicht, überprüfen Sie bitte die vorherigen Abschnitte und kehren Sie hierher zurück, sobald Sie diese Schritte abgeschlossen haben.

Bevor Sie einen neuen Minipool und Validator erstellen, sind einige Schritte erforderlich, um Ihren Node fertig vorzubereiten.
Diese müssen jedoch nur einmal durchgeführt werden; sobald Sie sie erledigt haben, können Sie zum Abschnitt [Erstellen eines neuen Minipools](./create-validator.mdx) springen, wenn Sie mehrere Minipools auf Ihrem Node erstellen möchten.

## Ihr Node-Wallet aufladen

Die Registrierung Ihres Nodes und das Aufsetzen eines Validators erfordern beide das Absenden von Transaktionen an das Ethereum-Netzwerk von Ihrem Node-Wallet aus.
Das bedeutet, **Sie müssen etwas ETH darauf haben**, um die Gaskosten für diese Transaktionen zu bezahlen.
Sie müssen auch einige **RPL-Token** als Sicherheit vor dem Erstellen eines Minipools staken; Sie können dies direkt auf dem Node tun, oder (vorzugsweise) Sie können die **Stake on Behalf**-Funktion der Rocket Pool-Website verwenden, um für Ihren Node mit RPL in Ihrem Cold Wallet zu staken.
Wir werden die Stake on Behalf-Funktion später in diesem Leitfaden besprechen, wenn es Zeit ist, Ihr RPL zu staken.

Das für Gasgebühren benötigte ETH zum Einrichten eines Nodes mit einem Minipool beträgt etwa 0,0025 ETH mal dem aktuellen Gaspreis in gwei.
Zum Beispiel würden Sie bei einem Gaspreis von 30 gwei etwa 0,075 ETH an Gasgebühren zahlen.

Sehen Sie sich [dieses Arbeitsblatt](https://docs.google.com/spreadsheets/d/1loB9U8wCIJn73a1DsR-mJJ9PQaYWMlF9asNc7BgM_zc/edit#gid=1413609552https://docs.google.com/spreadsheets/d/1loB9U8wCIJn73a1DsR-mJJ9PQaYWMlF9asNc7BgM_zc/edit#gid=1413609552) an, um die Gasgebühren für verschiedene Netzwerkbedingungen zu schätzen.

## Registrierung Ihres Nodes im Netzwerk

Sobald Sie ETH in Ihrem Wallet haben, können Sie Ihren Node im Rocket Pool-Netzwerk registrieren, um Zugriff auf alle Funktionen zu erhalten.
Führen Sie dazu den folgenden Befehl aus:

```shell
rocketpool node register
```

Sie werden nach der Zeitzone gefragt, mit der Sie sich registrieren möchten.
Standardmäßig wird diese von Ihrer Systemuhr erkannt, aber Sie können sie ändern, wenn Sie möchten.
Alle Zeitzonen im `Land/Stadt`-Format, die [auf dieser Seite aufgeführt sind](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones), sollten akzeptabel sein.

::: tip HINWEIS
Die Zeitzone wird nur für die globale Karte der Node-Betreiber auf der Hauptseite verwendet.
Sie müssen sie nicht auf Ihre tatsächliche Zeitzone einstellen, wenn Sie Sicherheitsbedenken haben.
Wenn Sie lieber anonym bleiben möchten, verwenden Sie eine generische Option wie `Etc/UTC`.
:::

Sobald dies abgeschlossen ist, sind Sie offiziell Mitglied des Rocket Pool-Netzwerks!

## Ihre primäre Auszahlungsadresse festlegen

Vor allem anderen empfehlen wir dringend, die **primäre Auszahlungsadresse** für Ihren Node zu ändern.
Dies ist die Adresse, an die alle Ihre RPL-Checkpoint-Belohnungen, Ihr gestaktes RPL und Ihr Beacon Chain ETH gesendet werden, wenn Sie Ihre Checkpoint-Belohnungen beanspruchen oder Ihren Validator beenden und aus Ihrem Minipool abheben.

Optional können Sie eine Auszahlungsadresse für Ihr ETH und [eine neue für Ihr RPL](https://rpips.rocketpool.net/RPIPs/RPIP-31) festlegen. Die RPL-Auszahlungsadresse kann, falls festgelegt, RPL-Inflationsbelohnungen auslösen und beanspruchen und hat keine Auswirkungen auf ETH-Konsensbelohnungen oder irgendetwas im Zusammenhang mit ETH. Wenn Sie keine RPL-Auszahlungsadresse festgelegt haben, erhält die primäre Auszahlungsadresse auch RPL-Belohnungen. Standardmäßig **kann, wer die primäre Auszahlungsadresse kontrolliert, auf ALL Ihr gestaktes RPL, ETH und Belohnungen zugreifen.**

Die Adresse, die verwendet wird, um einen Node mit ETH zu versorgen, sollte die primäre Auszahlungsadresse für den Node sein. Klicken Sie [hier](../legacy/houston/stake-eth-on-behalf.mdx), um über **Staking von ETH im Namen eines Nodes** zu lesen.

::: warning HINWEIS
Bei der erstmaligen Einrichtung Ihres Nodes ist diese auf die Adresse Ihres Node-Wallets eingestellt.
Aus Sicherheitsgründen ist es jedoch **von entscheidender Bedeutung, diese auf eine andere Adresse zu setzen, die von einem Cold Wallet kontrolliert wird.**
Ein Cold Wallet könnte ein Hardware-Wallet sein (wie [Ledger](https://www.ledger.com/), [Trezor](https://trezor.io/) oder [Grid+](https://gridplus.io/)) oder ein Smart Contract Wallet (wie [Argent](https://www.argent.xyz/)).

Wir **empfehlen dringend**, dass Sie KEIN Hot Wallet wie MetaMask als primäre Auszahlungsadresse verwenden.

Auf diese Weise erhält der Angreifer, falls Ihr Node-Wallet kompromittiert wird, keinen Zugriff auf Ihr gestaktes ETH und RPL, indem er Sie zum Ausstieg zwingt, da all diese Gelder an Ihr separates Cold Wallet gesendet werden (das sie hoffentlich nicht haben).

Primäre Auszahlungsadressen werden auf Node-Betreiber-Ebene festgelegt.
Wenn Sie mehrere Minipools erstellen, beziehen sich alle auf dieselbe primäre Auszahlungsadresse, sodass Sie diese Einrichtung nur einmal durchführen müssen.

[Hier finden Sie](https://ethereum.org/en/security/#wallet-security) einige Grundlagen zur Wallet-Sicherheit von der Ethereum Foundation.
:::

Es gibt zwei verschiedene Möglichkeiten, dies zu tun.
Bitte lesen Sie beide Optionen unten, um festzustellen, welche auf Sie zutrifft.

## Ihre Snapshot-Signalisierungsadresse festlegen

Das Festlegen Ihrer Snapshot-Signalisierungsadresse ermöglicht es Ihnen, an Snapshot-Abstimmungen auf einem Browser oder Mobilgerät teilzunehmen, ohne Node-Schlüssel einem Hot Wallet aussetzen zu müssen.

Es gibt ein paar Dinge vorzubereiten:

- Die Adresse Ihres Nodes
- Eine Adresse, die Sie für Snapshot-Abstimmungen verwenden möchten (Signalisierungsadresse)

Sie signieren eine Nachricht, die besagt, dass die Adresse Ihres Nodes an die neue Adresse delegieren kann. Diese Nachricht drückt Ihre Absicht aus, Ihre Wallet-Adresse als Snapshot-Signalisierungsadresse zu verwenden.

Wählen Sie aus einem der folgenden Tabs aus, welches Netzwerk Sie verwenden.

<div className="p-3">
  <Tabs>
    <Tab label="Hoodi-Testnetzwerk">Wenn Sie dies im Hoodi-Testnetzwerk ausprobieren, können Sie auf dieser Seite signieren: https://testnet.node.rocketpool.net/signalling-address</Tab>
    <Tab label="Mainnet">Wenn Sie bereit sind, dies im Mainnet zu konfigurieren, gehen Sie stattdessen hierher: https://node.rocketpool.net/signalling-address</Tab>
  </Tabs>
</div>

:::danger WARNUNG
Laden Sie den privaten Schlüssel Ihres Nodes nicht in ein Hot Wallet. Bitte wählen Sie ein anderes Konto als Ihre Signalisierungsadresse.
Nachdem Sie die Signalisierungsadresse festgelegt haben, können Sie damit auf Snapshot mit der Abstimmungskraft Ihres Nodes abstimmen.
:::

Beginnen Sie damit, **die Adresse, die Sie als Signalisierungsadresse verwenden möchten**, mit der Website über MetaMask, WalletConnect oder eine der anderen von der Website unterstützten Methoden zu verbinden. Ihnen wird dann dieser Dialog angezeigt, um Ihre Node-Adresse nachzuschlagen.

Als Nächstes geben Sie Ihre Node-Adresse ein und klicken auf die orangefarbene Schaltfläche „Find". Dies prüft, ob die Adresse ein registrierter Node ist, und führt Sie dann zum nächsten Schritt.

:::tip TIPP
**Stellen Sie sicher, dass Sie die richtige Node-Adresse haben, bevor Sie dies tun!** Wenn Sie Ihre Node-Adresse bestätigen müssen, können Sie sie schnell über die CLI mit dem Befehl `rocketpool node status` abrufen.
:::

Sobald Sie sich angemeldet und Ihre Node-Adresse bestätigt haben, sehen Sie Ihre **Snapshot-Signalisierungsadresse**. Sie sollte mit dem Konto übereinstimmen, mit dem Sie sich auf der Website angemeldet haben. Überprüfen Sie dies, bevor Sie fortfahren. Sobald Sie sicher sind, dass Sie mit dem gewünschten Konto angemeldet sind, klicken Sie auf die orangefarbene Schaltfläche „Sign Message". Sie sehen eine Aufforderung in Ihrer Wallet-Erweiterung, die folgende Nachricht zu signieren:

```
`snapshot signalling address` may delegate to me for Rocket Pool governance
```

Das Signieren kostet Sie kein Gas, aber das Festlegen schon. Nachdem Sie signiert haben, gibt Ihnen das Frontend einen Befehl zum Einfügen in den Smartnode. Fügen Sie ihn in die CLI Ihres Smartnodes ein und folgen Sie den angezeigten Schritten. Der Befehl sollte etwa so aussehen:

```shell
rocketpool pdao set-signalling-address
`snapshot signalling address`
`EIP712 signature`
```

Wenn Sie diese Nachricht in Ihrer CLI sehen, sind Sie fertig!

```
The node's signalling address was successfully set to `snapshot signalling address`
```

:::tip TIPP
Machen Sie sich keine Sorgen, wenn Sie die Website versehentlich schließen oder den Befehl verlieren. Sie können die Schritte einfach wiederholen und **erneut mit derselben Node-Adresse und Signalisierungsadresse signieren**. Das Frontend verwendet `signer.Signmessage()` aus der ethers-Bibliothek, was bedeutet, dass Ihre Signatur bei derselben Eingabe deterministisch ist.
Klicken Sie [hier](https://docs.ethers.org/v6/api/providers/#cid_865), um mehr zu erfahren.
:::

Das Löschen Ihrer Signalisierungsadresse ist ziemlich einfach, verwenden Sie einfach diesen Befehl in der CLI:

```shell
rocketpool pdao clear-signalling-address
```

## Ihre RPL-Auszahlungsadresse festlegen

Es ist wichtig, den Unterschied zwischen Ihrer [primären Auszahlungsadresse](./prepare-node#setting-your-primary-withdrawal-address) und Ihrer RPL-Auszahlungsadresse zu verstehen. Standardmäßig ist Ihre RPL-Auszahlungsadresse nicht festgelegt und Ihre primäre Auszahlungsadresse ist die, an die alle Ihre RPL-Checkpoint-Belohnungen, Ihr gestaktes RPL und Beacon Chain ETH gesendet werden.

Sie können eine [separate Auszahlungsadresse für Ihr RPL](https://rpips.rocketpool.net/RPIPs/RPIP-31) festlegen, wenn Sie möchten. Ihre RPL-Auszahlungsadresse kann, falls festgelegt, RPL-Inflationsbelohnungen auslösen und beanspruchen und hat keine Auswirkungen auf ETH-Konsensbelohnungen oder irgendetwas im Zusammenhang mit ETH. In diesem Fall hat Ihre primäre Auszahlungsadresse keine Autorität mehr über RPL-bezogene Aktionen.

Dies schafft einige interessante Möglichkeiten, bei denen RPL von einer Entität einem Node-Betreiber bereitgestellt werden kann, der kein Exposure zu RPL wünscht. Diese Entität kann dann RPL-Belohnungen für die Bereitstellung der erforderlichen Versicherungssicherheit für den Node beanspruchen. Mit anderen Worten, dies ermöglicht es verschiedenen Akteuren, jeden Sicherheitentyp bereitzustellen.

::: warning HINWEIS
Bei der erstmaligen Einrichtung Ihres Nodes ist Ihre RPL-Auszahlungsadresse standardmäßig nicht festgelegt.
Wenn Sie sich entscheiden, eine RPL-Auszahlungsadresse festzulegen, **ist es von entscheidender Bedeutung, diese auf eine andere Adresse zu setzen, die von einem Cold Wallet kontrolliert wird.**
Ein Cold Wallet könnte ein Hardware-Wallet sein (wie [Ledger](https://www.ledger.com/), [Trezor](https://trezor.io/) oder [Grid+](https://gridplus.io/)) oder ein Smart Contract Wallet (wie [Safe](https://app.safe.global/welcome)).

Wir **empfehlen dringend**, dass Sie KEIN Hot Wallet wie MetaMask als RPL-Auszahlungsadresse verwenden.

Auf diese Weise erhält der Angreifer, falls Ihr Node-Wallet kompromittiert wird, keinen Zugriff auf Ihr gestaktes RPL, indem er Sie zum Ausstieg zwingt, da all diese Gelder an Ihr separates Cold Wallet gesendet werden (das sie hoffentlich nicht haben).

RPL-Auszahlungsadressen werden auf Node-Betreiber-Ebene festgelegt.
Wenn Sie mehrere Minipools erstellen, beziehen sich alle auf dieselbe RPL-Auszahlungsadresse, sodass Sie diese Einrichtung nur einmal durchführen müssen.

[Hier finden Sie](https://ethereum.org/en/security/#wallet-security) einige Grundlagen zur Wallet-Sicherheit von der Ethereum Foundation.
:::

## Einen ENS-Namen Ihrem Node-Wallet zuordnen

Wenn Sie möchten, dass Ihre Node-Wallet-Adresse einem für Menschen lesbaren Namen wie `alice.eth` zugeordnet wird, befolgen Sie diese Schritte:

1. Kaufen Sie einen ENS-Namen auf [der offiziellen ENS-Website](https://app.ens.domains).

2. Richten Sie immer noch auf der ENS-Website die **Vorwärtsauflösung** für den Namen ein und definieren Sie, welche Adresse verwendet wird, wenn jemand auf Ihren `.eth`-Namen verweist.

- Wählen Sie „My account"
- Klicken Sie auf den ENS-Namen, den Sie bearbeiten möchten
- Klicken Sie auf „Add/edit record"
- Geben Sie Ihre gewünschte ETH-Adresse ein, klicken Sie auf „Confirm" und senden Sie die Transaktion ab.

3. Nachdem die vorherige Transaktion bestätigt wurde, können Sie die Smartnode-CLI verwenden, um die **Rückwärtsauflösung** einzurichten. Dies ermöglicht es Anwendungen, von Ihrer Node-Wallet-Adresse zurück auf Ihren `.eth`-Namen zu mappen.

Führen Sie den Befehl aus:

```shell
rocketpool wallet set-ens-name <your-ens-name.eth>
```

Diese Aktion erfordert auch das Senden einer Transaktion. Sie müssen die gewünschten Parameter bestätigen und die Transaktion absenden.

::: warning HINWEIS
**Stellen Sie sicher, dass Sie Schritt 2 abgeschlossen haben, um die Vorwärtsauflösung einzurichten, bevor Sie den CLI-Befehl zum Einrichten der Rückwärtsauflösung ausführen**. Dies ist eine Sicherheitsmaßnahme, um zu verhindern, dass Wallets versuchen, einen ENS-Namen nachzuahmen, den sie nicht kontrollieren.
:::

4. Nachdem die Transaktion erfolgreich in einen Block aufgenommen wurde, führen Sie `rocketpool node status` aus, um zu sehen, dass Ihr Wallet jetzt mit Ihrem ENS-Namen verknüpft ist.
