# Das Saturn-0-Upgrade

Willkommen bei Saturn 0! Dieses Upgrade zielt darauf ab, die RPL-Tokenomics kurzfristig zu verbessern, um das Protokollwachstum freizuschalten. Wenn Sie ein wiederkehrender oder potenzieller Node-Betreiber sind, dient diese Seite als Einführung in einige neue und aufregende Änderungen am Rocket Pool-Protokoll.

## Nur-ETH-Minipools in Saturn 0

- Die RPL-Anforderung für neue Minipools gehört der Vergangenheit an! Minipools in Saturn 0 benötigen keinen RPL-Stake, um bereitgestellt zu werden.
- Die RPL-Anforderung für neue Minipools gehört der Vergangenheit an! Minipools in Saturn 0 benötigen keinen RPL-Stake, um bereitgestellt zu werden.
  Node-Betreiber können jetzt Minipools mit Vorabkosten von 8 ETH erstellen, die eine Provision von 10 % aufweisen. Darüber hinaus kann ein Node-Betreiber optional RPL staken, um sich für zusätzliche Belohnungen zu qualifizieren und Zugang zu Stimmrecht innerhalb der [Protocol DAO](/de/legacy/houston/pdao#the-protocol-dao-pdao) zu erhalten.

- Nodes mit einem RPL-Stake jeder Größe qualifizieren sich für RPL-Emissionsbelohnungen. Dies ermöglicht RPL-Belohnungen für Node-Betreiber unter der 10%-Sicherheitsgrenze, die zuvor nicht für RPL-Emissionsbelohnungen qualifiziert waren.

- Das bedeutet, dass Nodes mit einem RPL-Stake von 0-15 % des geliehenen ETH den maximalen APY auf RPL erhalten. Nodes verdienen immer noch RPL-Rendite über 15 % Sicherheit, mit abnehmender Rate.
  Schauen Sie sich gerne an, wie die RPL-Emission über 15 % geliehenem ETH in [RPIP-30](https://rpips.rocketpool.net/RPIPs/RPIP-30#specification) definiert ist.

## Neue Provisionsstruktur

Wir freuen uns, ankündigen zu können, dass neu gestartete Saturn-0-Minipools eine Provisionsstruktur von 10 % bis 14 % aufweisen werden.

- Nodes ohne gestaktes RPL verdienen 10 % Provision (5 % aus dem Minipool-Contract und 5 % aus dem dynamischen Provisionsbonus).
- Nodes mit gestaktem RPL erhalten bis zu zusätzliche 4 % dynamischen Provisionsbonus für gestaktes RPL im Wert von >= 10 % des geliehenen ETH.

Beachten Sie, dass der dynamische Provisionsbonus beim 5. Belohnungs-Snapshot nach der Veröffentlichung von Saturn 1 auf 0 % reduziert wird (so dass nur die 5 % Minipool-Contract-Provision übrig bleibt). Dies dient dazu, den Renditevorteil der Migration zu Megapools zu erhöhen.

::: warning WARNUNG
Ihr Node muss in den [Smoothing Pool](/de/node-staking/fee-distrib-sp#the-smoothing-pool) aufgenommen sein, um vom dynamischen Provisionsbonus zu profitieren. Der Smart Node wird Sie darüber informieren, sich beim Einzahlen eines Minipools in den Smoothing Pool aufzunehmen, falls Sie noch nicht aufgenommen sind. _Ihre neuen Minipools werden zusätzliche Rendite verpassen, wenn Ihr Node nicht im Smoothing Pool ist!!_
:::

## Node-Betreiber-Leitfaden

Wenn Sie daran interessiert sind, einen Rocket Pool-Node zu betreiben, um Ihre eigenen Ethereum-Validators zu erstellen, werfen Sie einen Blick auf unseren [Leitfaden für Node-Betreiber](/de/node-staking/responsibilities).
Dies ist ein sehr umfassender Leitfaden, der alle Überlegungen zum Node-Betrieb durchgeht, einschließlich:

- Entscheidung, ob ein Node lokal zu Hause oder remote in der Cloud betrieben werden soll
- Auswahl der Hardware für eine lokale Maschine oder Auswahl eines Remote-Hosting-Anbieters
- Vorbereitung und Sicherung des Betriebssystems Ihres Node
- Auswahl Ihrer Execution- und Consensus-Clients
- Installation des Smartnode-Stacks von Rocket Pool
- Erstellung eines neuen Beacon-Chain-Validators mit dem Rocket Pool-Netzwerk
- Wartung und Aktualisierung Ihres Node
- Teilnahme an On-Chain-Governance

Beide Leitfäden zeigen Ihnen, wie Sie kostenlos im Rocket Pool-Testnet üben können, damit Sie ein Gefühl für alles bekommen, ohne Ihr echtes ETH zu riskieren, und wie Sie Ihr echtes ETH auf dem Ethereum-Mainnet staken, sobald Sie mit dem Prozess vertraut sind.

Wenn Sie bereit sind, wählen Sie einen Leitfaden aus den oben genannten Optionen basierend auf Ihren Interessen aus.
Sie können auch die verschiedenen Seiten über die Seitenleiste auf der linken Seite erkunden.
