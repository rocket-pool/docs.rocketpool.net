# Festlegen Ihrer RPL-Abhebungsadresse

Es ist wichtig, den Unterschied zwischen Ihrer [primären Abhebungsadresse](../../node-staking/prepare-node#setting-your-primary-withdrawal-address) und Ihrer RPL-Abhebungsadresse zu verstehen. Standardmäßig ist Ihre RPL-Abhebungsadresse nicht festgelegt und Ihre primäre Abhebungsadresse ist die Adresse, an die alle Ihre RPL-Checkpoint-Belohnungen, Ihr gestaketes RPL und Beacon Chain ETH gesendet werden.

Sie können auf Wunsch eine [separate Abhebungsadresse für Ihr RPL](https://rpips.rocketpool.net/RPIPs/RPIP-31) festlegen. Ihre RPL-Abhebungsadresse, falls festgelegt, kann RPL-Inflationsbelohnungen auslösen und einfordern und hat keine Auswirkungen auf ETH-Konsensbelohnungen oder irgendetwas mit ETH zusammenhängendes. In diesem Fall hat Ihre primäre Abhebungsadresse keine Autorität mehr über RPL-bezogene Aktionen.

Dies schafft einige interessante Möglichkeiten, bei denen RPL von einer Entität einem Node-Operator bereitgestellt werden kann, der keine Exposition gegenüber RPL wünscht. Diese Entität kann dann RPL-Belohnungen dafür einfordern, dass sie die erforderliche Versicherungssicherheit für den Node bereitstellt. Mit anderen Worten, dies ermöglicht es verschiedenen Akteuren, jede Sicherheitenart bereitzustellen.

::: warning HINWEIS
Bei der erstmaligen Einrichtung Ihres Nodes ist Ihre RPL-Abhebungsadresse standardmäßig nicht festgelegt.
Wenn Sie sich entscheiden, eine RPL-Abhebungsadresse festzulegen, **ist es von entscheidender Bedeutung, diese auf eine andere Adresse zu setzen, die von einem Cold Wallet kontrolliert wird.**
Ein Cold Wallet könnte ein Hardware-Wallet sein (wie ein [Ledger](https://www.ledger.com/), [Trezor](https://trezor.io/) oder [Grid+](https://gridplus.io/)) oder ein Smart Contract Wallet (wie [Safe](https://app.safe.global/welcome)).

Wir **empfehlen dringend**, dass Sie KEIN Hot Wallet wie MetaMask als Ihre RPL-Abhebungsadresse verwenden.

Auf diese Weise erhält ein Angreifer, falls Ihr Node-Wallet kompromittiert wird, keinen Zugriff auf Ihr gestaketes RPL, indem er Sie zum Austritt zwingt, da all diese Mittel an Ihr separates Cold Wallet gesendet werden (auf das er hoffentlich keinen Zugriff hat).

RPL-Abhebungsadressen werden auf Node-Operator-Ebene festgelegt.
Wenn Sie mehrere Minipools erstellen, beziehen sie sich alle auf dieselbe RPL-Abhebungsadresse, sodass Sie diese Einrichtung nur einmal durchführen müssen.

[Siehe hier](https://ethereum.org/en/security/#wallet-security) für einige Grundlagen zur Wallet-Sicherheit von der Ethereum Foundation.
:::

Es gibt zwei verschiedene Möglichkeiten, dies zu tun.
Bitte lesen Sie beide Optionen unten, um zu bestimmen, welche auf Sie zutrifft.

Sobald dies erledigt ist, können Sie **Ihre RPL-Abhebungsadresse nicht mehr mit dem Befehl `set-rpl-withdrawal-address` ändern**.
Um sie zu ändern, müssen Sie eine signierte Transaktion von Ihrer **aktiven** RPL-Abhebungsadresse (derjenigen, zu der Sie gerade gewechselt haben) senden.
Die Rocket Pool-Website verfügt über eine Funktion, die Ihnen dabei hilft.

Wenn Sie Ihre RPL-Abhebungsadresse aufheben möchten, ist der Prozess ziemlich einfach. Zuerst möchten Sie zurück zur RPL-Abhebungsadresse-Seite auf der Rocket Pool-Website navigieren.
