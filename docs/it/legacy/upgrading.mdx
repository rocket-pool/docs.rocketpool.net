# Migrazione dello Smartnode dai Test Beta Precedenti

Se sei stato un Node Operator di Rocket Pool durante i nostri precedenti test beta su Medalla o Pyrmont, grazie per averci aiutato a migliorare il protocollo e bentornato!
Questa breve guida ti accompagnerà nel processo di eliminazione della tua build beta e nella configurazione della build più recente per la testnet Hoodi o per lo staking reale su mainnet.

      Ci sono tre passaggi principali per l'aggiornamento da una build beta alla rete di test Hoodi.

      Prima, migra qualsiasi hoodi ETH che possiedi in un wallet separato.
      Potrai quindi migrarlo nuovamente nel nuovo wallet del nodo che creerai dopo aver reinstallato Rocket Pool, così non dovrai fare troppo affidamento su un faucet per l'hoodi ETH.

      Il modo più semplice per farlo è tramite Metamask - vedi la [guida allo staking di rETH](../liquid-staking/overview) per una procedura su come configurare un nuovo wallet.
      Puoi farlo utilizzando il seguente comando:

      ```shell
      rocketpool node send XXX eth 0xabcd1234...
      ```

      dove `XXX` è la quantità di hoodi ETH da inviare, e `0xabcd1234...` è l'indirizzo del nuovo wallet che hai creato.
      **Nota che dovrai lasciare un po' di ETH per pagare il gas utilizzato nella transazione!**

      Successivamente, rimuovi tutte le vecchie immagini Docker e i dati della chain con il seguente comando:

      ```shell
      rocketpool service terminate
      ```

      Una volta fatto, rimuovi i vecchi dati e la cartella delle impostazioni di Rocket Pool.
      **Questo eliminerà il tuo vecchio wallet e i tuoi vecchi validatori, quindi assicurati di non averne più bisogno per nulla!**

      ```shell
      sudo rm -rf ~/.rocketpool
      ```

      Una volta fatto, la tua vecchia configurazione beta sarà pulita.
      Puoi seguire la [guida all'installazione di Rocket Pool](../node-staking/eth-clients) per una procedura sull'installazione e la configurazione di Rocket Pool per la testnet Hoodi.
