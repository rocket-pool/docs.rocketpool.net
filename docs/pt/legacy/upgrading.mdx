# Migrando o Smartnode de Testes Beta Anteriores

Se você foi um operador de nó Rocket Pool durante nossos testes beta anteriores em Medalla ou Pyrmont, obrigado por nos ajudar a melhorar o protocolo e bem-vindo de volta!
Este breve guia o orientará no processo de limpeza de sua própria compilação beta e configuração da mais nova compilação para a testnet Hoodi ou para staking real na mainnet.

<div className="p-3">
  <Tabs>
    <Tab label="Rede Hoodi">
      <p className="rspress-directive warning">
        <p className="rspress-directive-title">NOTA</p>
        Você precisará criar uma nova carteira de nó para interagir com a rede de teste Hoodi.
        Reutilizar uma carteira de uma compilação beta anterior pode resultar em falhas ao criar novos validadores!
      </p>

      Existem três etapas principais para atualizar de uma compilação beta para a rede de teste Hoodi.

      Primeiro, migre qualquer ETH hoodi que você tenha para uma carteira separada.
      Você pode migrá-lo de volta para a nova carteira de nó que criar depois de reinstalar o Rocket Pool, para não precisar depender tanto de uma faucet de ETH hoodi.

      A maneira mais fácil de fazer isso é via Metamask - consulte o [guia de staking rETH](../liquid-staking/overview) para um passo a passo de como configurar uma nova carteira.
      Você pode fazer isso usando o seguinte comando:

      ```shell
      rocketpool node send XXX eth 0xabcd1234...
      ```

      onde `XXX` é a quantidade de ETH hoodi a enviar, e `0xabcd1234...` é o endereço da nova carteira que você criou.
      **Observe que você precisará deixar um pouco de ETH para pagar o gás usado na transação!**

      Em seguida, remova todas as imagens antigas do Docker e dados da chain com o seguinte comando:

      ```shell
      rocketpool service terminate
      ```

      Depois disso, remova a pasta antiga de dados e configurações do Rocket Pool.
      **Isso excluirá sua carteira antiga e seus validadores antigos, então certifique-se de que não precisa mais salvá-los para nada!**

      ```shell
      sudo rm -rf ~/.rocketpool
      ```

      Feito isso, sua configuração beta antiga será limpa.
      Você pode seguir o [guia de instalação do Rocket Pool](../node-staking/eth-clients) para um passo a passo sobre como instalar e configurar o Rocket Pool para a testnet Hoodi.
