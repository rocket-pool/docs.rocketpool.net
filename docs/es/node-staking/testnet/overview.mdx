# Practicando con la Red de Prueba

Una **red de prueba** de Ethereum (`testnet` para abreviar) es una blockchain de Ethereum, funcionalmente idéntica a la blockchain estándar de Ethereum, pero está destinada a ser utilizada para probar contratos inteligentes antes de que entren en funcionamiento en la blockchain "real" principal (llamada `mainnet`).
Las testnets funcionan en paralelo con mainnet - ¡en realidad hay varias de ellas funcionando ahora mismo!

Puedes leer más sobre algunas de las testnets de la Capa de Ejecución [aquí](https://support.mycrypto.com/how-to/getting-started/where-to-get-testnet-ether) si lo deseas.
Rocket Pool utiliza **Hoodi** para su testnet de Capa de Ejecución.
Puedes usar un explorador de bloques para ella aquí si deseas echarle un vistazo: [https://hoodi.etherscan.io/](https://hoodi.etherscan.io/)

El mismo concepto se aplica también a la Capa de Consenso.
Rocket Pool utiliza **Hoodi** para su testnet de Capa de Consenso.
Puedes encontrar un explorador de bloques para ella aquí: [https://hoodi.beaconcha.in/](https://hoodi.beaconcha.in/)

Debido a que las redes están destinadas a pruebas, **el ETH en ellas se entrega de forma gratuita para ayudar a las personas a probar**.
Este ETH **no puede transferirse** a mainnet; debe permanecer en Hoodi y no tiene ningún valor real, por lo que puedes pensar en él como **dinero del Monopoly**.

::: danger
**¡NO uses tu ETH real de mainnet en ningún momento si estás probando la red de prueba de Rocket Pool!**
¡**Perderás tus fondos** para siempre si haces esto!
¡Solo debes usar ETH de prueba que hayas recibido en la red Hoodi de un grifo (descrito a continuación)!
:::

## Configurando la Testnet

[La configuración predeterminada de Rocket Pool usando Docker](../install-modes) maneja automáticamente todas las configuraciones involucradas en el uso de una testnet por ti.
No hay nada especial que necesites hacer.

Si estás usando [una configuración diferente](../install-modes) con un cliente de ejecución o consenso externo, o estás ejecutando tus clientes [nativamente sin Docker](../install-modes), entonces necesitarás configurar esto manualmente:

- Para **Geth**: añade `--hoodi` al comando de inicio.
- Para **Lighthouse**: añade `--network hoodi` al comando de inicio.
- Para **Nimbus**: añade `--network=hoodi` al comando de inicio.
- Para **Prysm**: añade `--hoodi` al comando de inicio.
- Para **Teku**: añade `--network=hoodi` al comando de inicio.
- Para **Lodestar**: prefija el comando de inicio con `LODESTAR_NETWORK=hoodi`

Consulta la siguiente sección para aprender cómo obtener algo de ETH de prueba en Hoodi, que necesitarás para probar Rocket Pool.

## Obteniendo ETH de Prueba en Hoodi

Para usar la red de prueba de Rocket Pool, necesitarás ETH en Hoodi.
Por suerte, puedes obtener esto gratis usando un **grifo** - un servicio automatizado que te proporcionará ETH de Hoodi para probar.

::: warning NOTA
Necesitarás proporcionar una dirección de Ethereum para recibir ETH de prueba.

Si solo estás tratando de hacer stake con tu ETH normalmente y recibir rETH, por favor revisa la [Guía de Staking](../../liquid-staking/overview) antes de obtener ETH de Hoodi.
Se te pedirá que regreses aquí en el paso apropiado.

Si estás planeando ejecutar un nodo, por favor revisa la [Guía del Operador de Nodo](../responsibilities) para configurar un nodo y crear una nueva cartera primero.
Se te pedirá que regreses aquí en el paso apropiado.
:::

Rocket Pool proporciona un grifo en su [servidor de Discord](https://discord.gg/rocketpool).
Únete al servidor y dirígete al canal `#support`.

Aquí, puedes solicitar ETH de testnet.
Necesitarás proporcionar la dirección de tu nodo.

::: warning NOTA
El bot del grifo de testnet ya no es directamente accesible al público debido a abusos previos.
Solo está disponible para el equipo de desarrollo y miembros prominentes de la comunidad.
Para recibir eth de testnet, ejecuta los comandos a continuación, declara tus intenciones en el canal.
Ten paciencia y alguien accederá al grifo en tu nombre.
:::

Si no estás planeando registrar un nodo y solo quieres algo de ETH para probar el staking y recibir rETH, escribe el siguiente comando en el canal:

```
!hoodiEth <tu dirección de hoodi>
```

sin los paréntesis angulares.
Si tu cartera no tiene ningún ETH ya, el bot enviará a tu cartera 1 ETH para probar.

Si _estás_ planeando ejecutar un nodo, haz lo siguiente:

1. Comienza ejecutando el comando `!hoodiEth` anterior para adquirir suficiente ETH para registrar tu nodo
2. Registra tu nodo (consulta la página [Preparando tu Nodo](../prepare-node) para instrucciones)
3. Ejecuta el siguiente comando para obtener 8 ETH de Hoodi para que puedas crear un validador:

```
!hoodiOperator <tu dirección de hoodi>
```

Para usar `!hoodiOperator`, la dirección que proporciones _debe_ ser un nodo registrado de Rocket Pool.
Por lo tanto, debes seguir estos 3 pasos en ese orden.

## Obteniendo RPL de Prueba en Hoodi

Si estás probando la operación de nodos, necesitarás algo de RPL de prueba además de ETH de prueba.

Rocket Pool proporciona un grifo en su [servidor de Discord](https://discord.gg/rocketpool).
Únete al servidor y dirígete al canal `#support`.

Aquí, puedes solicitar RPL de testnet.
Necesitarás proporcionar la dirección de tu nodo.

::: warning NOTA
El bot del grifo de testnet ya no es directamente accesible al público debido a abusos previos.
Solo está disponible para el equipo de desarrollo y miembros prominentes de la comunidad.
Para recibir RPL de testnet, ejecuta los comandos a continuación, declara tus intenciones en el canal.
Ten paciencia y alguien accederá al grifo en tu nombre.
:::

```
!hoodiRpl <tu dirección de nodo de hoodi>
```

Esto te proporcionará 600 del token **RPL heredado (v1)**, que es análogo al token RPL original en mainnet.
Rocket Pool usa un **nuevo token RPL (v2)** que soporta la inflación utilizada para recompensar a los operadores de nodos.

Para aprender cómo intercambiar RPL heredado por RPL nuevo, sigue los pasos en la guía [Creando un Minipool (Validador)](../create-validator).
