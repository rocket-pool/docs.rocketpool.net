# Migración del Smartnode desde Pruebas Beta Anteriores

Si has sido un operador de nodos de Rocket Pool durante nuestras pruebas beta anteriores en Medalla o Pyrmont, gracias por ayudarnos a mejorar el protocolo y bienvenido de vuelta.
Esta breve guía te guiará a través del proceso de purgar tu compilación beta anterior y configurar la compilación más nueva ya sea para la testnet Hoodi o para staking real en mainnet.

<div className="p-3">
  <Tabs>
    <Tab label="Hoodi Network">
      <p className="rspress-directive warning">
        <p className="rspress-directive-title">NOTA</p>
        Necesitarás crear una nueva wallet de nodo para interactuar con la red de prueba Hoodi.
        Reutilizar una wallet de una compilación beta anterior puede resultar en fallas al crear nuevos validadores.
      </p>

      Hay tres pasos principales hacia la actualización desde una compilación beta a la red de prueba Hoodi.

      Primero, migra cualquier ETH de hoodi que tengas a una wallet separada.
      Luego puedes migrarlo de vuelta a la nueva wallet de nodo que crees después de reinstalar Rocket Pool, para que no necesites depender tanto de un faucet para ETH de hoodi.

      La forma más fácil de hacer esto es a través de Metamask - consulta la [guía de staking de rETH](../liquid-staking/overview) para un tutorial de cómo configurar una nueva wallet.
      Puedes hacer esto usando el siguiente comando:

      ```shell
      rocketpool node send XXX eth 0xabcd1234...
      ```

      donde `XXX` es la cantidad de ETH de hoodi a enviar, y `0xabcd1234...` es la dirección de la nueva wallet que creaste.
      **Ten en cuenta que necesitarás dejar un poco de ETH para pagar el gas usado en la transacción.**

      A continuación, elimina todas las imágenes Docker antiguas y los datos de cadena con el siguiente comando:

      ```shell
      rocketpool service terminate
      ```

      Una vez hecho esto, elimina la carpeta de datos y configuración antigua de Rocket Pool.
      **Esto eliminará tu wallet antigua y tus validadores antiguos, así que asegúrate de que ya no necesites guardarlos para nada.**

      ```shell
      sudo rm -rf ~/.rocketpool
      ```

      Una vez hecho esto, tu configuración beta antigua estará limpia.
      Puedes seguir la [guía de instalación de Rocket Pool](../node-staking/eth-clients) para un tutorial sobre cómo instalar y configurar Rocket Pool para la testnet Hoodi.
