# 이전 베타 테스트에서 Smartnode 마이그레이션

Medalla 또는 Pyrmont에서 이전 베타 테스트 동안 Rocket Pool 노드 운영자였다면, 프로토콜 개선에 도움을 주셔서 감사하며 다시 오신 것을 환영합니다!
이 간단한 가이드는 자신의 베타 빌드를 제거하고 Hoodi 테스트넷 또는 메인넷의 실제 스테이킹을 위한 최신 빌드를 설정하는 과정을 안내합니다.

<div className="p-3">
  <Tabs>
    <Tab label="Hoodi 네트워크">
      <p className="rspress-directive warning">
        <p className="rspress-directive-title">참고</p>
        Hoodi 테스트 네트워크와 상호 작용하려면 새 노드 지갑을 생성해야 합니다.
        이전 베타 빌드의 지갑을 재사용하면 새 검증자를 생성할 때 실패할 수 있습니다!
      </p>

      베타 빌드에서 Hoodi 테스트 네트워크로 업그레이드하는 세 가지 주요 단계가 있습니다.

      먼저, 보유하고 있는 hoodi ETH를 별도의 지갑으로 마이그레이션합니다.
      Rocket Pool을 재설치한 후 생성한 새 노드 지갑으로 다시 마이그레이션할 수 있으므로 hoodi ETH에 대해 파우셋에 많이 의존할 필요가 없습니다.

      가장 쉬운 방법은 Metamask를 사용하는 것입니다 - 새 지갑을 설정하는 방법에 대한 안내는 [rETH 스테이킹 가이드](../liquid-staking/overview)를 참조하세요.
      다음 명령을 사용하여 이 작업을 수행할 수 있습니다:

      ```shell
      rocketpool node send XXX eth 0xabcd1234...
      ```

      여기서 `XXX`는 보낼 hoodi ETH의 양이고, `0xabcd1234...`는 생성한 새 지갑의 주소입니다.
      **거래에 사용되는 가스를 지불하기 위해 약간의 ETH를 남겨두어야 합니다!**

      다음으로, 다음 명령으로 모든 이전 Docker 이미지와 체인 데이터를 제거합니다:

      ```shell
      rocketpool service terminate
      ```

      이 작업이 완료되면 이전 Rocket Pool 데이터 및 설정 폴더를 제거합니다.
      **이렇게 하면 이전 지갑과 이전 검증자가 삭제되므로 더 이상 저장할 필요가 없는지 확인하세요!**

      ```shell
      sudo rm -rf ~/.rocketpool
      ```

      이 작업이 완료되면 이전 베타 설정이 정리됩니다.
      Hoodi 테스트넷용 Rocket Pool을 설치하고 구성하는 방법에 대한 안내는 [Rocket Pool 설치 가이드](../node-staking/eth-clients)를 참조하세요.
