# Rocket Pool을 통해 직접 스테이킹

## 개요

Rocket Pool의 스테이킹 인터페이스는 이제 ETH와 rETH 간 변환을 위한 두 가지 별도의 경로를 제공합니다:

1. **프로토콜 경로** - Rocket Pool 스마트 컨트랙트와 직접 상호 작용
2. **CoW Swap 경로** - MEV 보호 기능이 있는 탈중앙화 거래소 통합기 (메인넷 전용)

인터페이스는 현재 시장 상황과 유동성 가용성을 기반으로 최상의 경로를 자동으로 권장합니다.

## 라우팅 옵션 설명

### 프로토콜 경로

**작동 방식:** 온체인에서 Rocket Pool의 예치 풀 및 스마트 컨트랙트와 직접 상호 작용합니다.

**장점:**

- 즉시 실행
- 현재 rETH/ETH 비율을 기반으로 한 예측 가능한 환율
- 중개 토큰 불필요 (직접 ETH ↔ rETH 변환)
- 낮은 복잡성
- 항상 ETH의 가치만큼 정확히 rETH를 받습니다 (0.05% 예치 수수료 제외)
- 슬리피지, 유동성 문제, 샌드위칭 또는 프런트러닝 없음

**제한사항:**

- 스테이킹 시 예치 풀 한도의 적용을 받음
- 언스테이킹 시 유동성 가용성의 적용을 받음
- 거래에 가스 수수료 필요
- 소액의 경우 가스 수수료가 비쌀 수 있음

### CoW Swap 경로 (메인넷 전용)

**작동 방식:** CoW 프로토콜의 배치 경매 시스템을 사용하여 여러 DEX에서 최상의 가격을 찾습니다.

**장점:**

- 샌드위치 공격에 대한 MEV(Maximum Extractable Value) 보호
- 가스 없는 실행 - 주문에 서명만 하면 됩니다
- DEX 통합을 통해 종종 더 나은 비율 제공
- 프로토콜 예치 풀 한도와 관계없이 항상 사용 가능
- 스왑 자체에 대한 가스 수수료 없음

**제한사항:**

- WETH 래핑/언래핑 단계 필요 (자동으로 처리됨)
- 주문 실행에 2-5분 소요될 수 있음
- 제한 시간 내에 매칭되지 않으면 주문이 만료될 수 있음
- 일회성 토큰 승인 필요

## 주요 개념

### WETH (Wrapped ETH)

CoW Swap을 사용할 때 인터페이스는 자동으로 WETH 변환을 처리합니다:

- WETH는 표준화된 형식으로 ETH를 나타내는 ERC-20 토큰입니다
- 1 WETH는 항상 1 ETH와 같습니다
- 인터페이스는 거래 전에 ETH를 WETH로 래핑하고 거래 후 언래핑합니다
- 초기 래핑에 대한 가스만 지불하면 됩니다 (일회성)

### 슬리피지 허용 범위

CoW Swap 주문의 경우:

- 기본값: 0.5%
- 선호도에 따라 조정 가능
- 슬리피지가 높을수록 = 실행 가능성이 높아짐
- 슬리피지가 낮을수록 = 더 나은 가격 보호

### 주문 만료

CoW Swap 주문에는 시간 제한이 있습니다:

- 기본값: 30분
- 5분에서 24시간까지 조정 가능
- 주문이 실행되지 않으면 자동으로 취소됨

### 더 나은 비율 표시기

인터페이스는 어떤 경로가 더 나은 가치를 제공하는지 보여줍니다:

- "X% 더 나음"이 있는 녹색 배지는 최적의 경로를 나타냅니다
- 최상의 옵션을 자동으로 미리 선택합니다
- 시장 상황이 변경됨에 따라 실시간으로 업데이트됩니다

### 프로토콜 유동성

rETH를 언스테이킹할 때:

- 프로토콜 경로는 예치 풀에 사용 가능한 ETH가 필요합니다
- 풀이 비어 있을 때 "유동성 부족"이 표시됩니다
- 프로토콜 유동성을 사용할 수 없을 때 자동으로 CoW Swap을 통해 라우팅됩니다

## ETH를 rETH로 스테이킹

스테이킹 사이트에 도착하면 **connect wallet** 버튼을 클릭하세요. 서비스 약관 및 개인정보 보호정책을 읽고 수락하면 다양한 연결 방법이 활성화됩니다. 그런 다음 **connect metamask**를 클릭하세요.

MetaMask는 웹사이트에 연결할 계정을 선택하라는 메시지를 표시합니다. 하나를 선택하고 몇 가지 권한을 확인하면 UI에서 잔액이 업데이트되는 것을 볼 수 있습니다.

### 1단계: 금액 입력

**Stake ETH** 입력 필드에 스테이킹하려는 ETH 금액을 입력하세요.

[이미지 자리 표시자: 금액이 입력되는 입력 필드]

### 2단계: 라우팅 옵션 검토

인터페이스는 두 가지 라우팅 옵션을 표시합니다 (메인넷 전용):

- **프로토콜 카드**: 예치 풀이 사용 가능한지 가득 찼는지 표시합니다
- **CoW Swap 카드**: 예상 출력 및 비율 비교를 표시합니다

더 나은 옵션은 녹색 "X% 더 나음" 배지로 강조 표시되고 자동으로 선택됩니다.

[이미지 자리 표시자: 비교를 보여주는 라우팅 카드]

### 3단계: 설정 구성 (CoW Swap 전용)

CoW Swap을 사용하는 경우 선택적으로 다음을 수행할 수 있습니다:

- 슬리피지 허용 범위 조정 (기본값 0.5%)
- 주문 만료 시간 설정 (기본값 30분)

## 문제 해결

### 일반적인 문제

#### 주문 만료됨 (CoW Swap)

제한 시간 내에 주문이 실행되지 않았습니다.

**해결 방법:**

- 주문 만료 시간 늘리기 (최대 24시간)
- 슬리피지 허용 범위를 높게 조정
- 현재 시장 가격으로 다시 시도
- 사용 가능한 경우 프로토콜 경로 사용

#### 잔액 부족

거래에 필요한 ETH/rETH가 충분하지 않습니다.

이 섹션은 일반적인 교육 정보만 제공합니다. 세법은 관할 구역 및 개인 상황에 따라 크게 다릅니다. 귀하의 상황에 특정한 조언은 **항상 자격을 갖춘 세무 전문가와 상담하세요**.

### rETH와 세금 이해

Rocket Pool로 스테이킹할 때 잠재적인 세금 영향을 이해하는 것이 중요합니다:

#### 스테이킹 거래 (ETH → rETH)

**잠재적 과세 사건:**

- **토큰 스왑**: 많은 관할 구역에서 ETH를 rETH로 변환하는 것이 과세 사건으로 간주될 수 있습니다
- **자본 이득/손실**: 취득 이후 ETH가 가치가 상승했다면 자본 이득세가 부과될 수 있습니다
- **취득 원가**: 받은 rETH의 가치가 향후 거래를 위한 새로운 취득 원가가 됩니다

**예시 시나리오:**

```
$1,000에 1 ETH 구매
ETH가 $2,000일 때 스테이킹
잠재적 과세 이득: $1,000
rETH 취득 원가: $2,000
```

#### rETH 보유 (보상 획득)

**rETH가 가치 상승하는 방법:**

- rETH는 ETH에 비해 자동으로 가치가 증가합니다
- 새로운 토큰을 받지 않음 - 토큰 자체가 가치 상승합니다
- 이 가치 상승은 스테이킹 보상을 나타냅니다

**세금 처리 방식은 다양합니다:**

- **일부 관할 구역**: 실현될 때만 가치 상승에 대한 세금 (판매/스왑 시)
- **기타 관할 구역**: 미실현 이득에 대해 매년 과세할 수 있음
- **소득 대 자본 이득**: 분류가 세율에 영향을 미칩니다

#### 언스테이킹 거래 (rETH → ETH)

**세금 고려 사항:**

- rETH를 ETH로 다시 변환하는 것은 일반적으로 과세 사건입니다
- 다음을 기반으로 이득/손실 계산:
- **취득 원가**: rETH를 취득했을 때의 가치
- **처분 가치**: ETH로 다시 변환할 때의 가치
- 차이가 자본 이득 또는 손실입니다

### 라우팅 방법 세금 영향

#### 프로토콜 경로

- 거래당 단일 과세 사건
- 세금 기록을 위한 명확한 타임스탬프 및 가치
- 온체인 거래는 영구 기록을 제공합니다

#### CoW Swap 경로

- WETH 래핑/언래핑은 일반적으로 과세되지 않음 (1:1 변환)
- 주요 스왑 (ETH/WETH ↔ rETH)이 과세 사건입니다
- 주문 실행은 명확한 세금 문서를 제공합니다

### 세금 최적화 전략

**타이밍 고려 사항:**

- **장기 대 단기**: 많은 관할 구역에서 장기 보유 선호 (일반적으로 >1년)

**취득 원가 방법:**

- **LIFO (Last In, First Out)**: 일부 지역에서 사용 가능할 수 있음
- **특정 식별**: 허용되는 경우 특정 rETH 부분 추적

## FAQ

**Q: 어떤 경로를 선택해야 하나요?**
A: 인터페이스는 녹색 "더 나은 비율" 배지로 최상의 경로를 자동으로 권장합니다. 일반적으로 권장 옵션을 사용하세요.

**Q: CoW Swap이 때때로 더 나은 비율을 제공하는 이유는 무엇인가요?**
A: CoW Swap은 여러 DEX의 유동성을 통합하고 배치 경매를 사용하여 최적의 가격을 찾으며, 종종 고정 프로토콜 비율을 능가합니다.

**Q: CoW Swap이 정말 가스 없나요?**
A: 주문 생성 및 서명은 가스 없이 이루어집니다. 초기 WETH 래핑 및 토큰 승인에 대해서만 가스를 지불합니다 (일회성 설정).

**Q: CoW Swap 주문은 얼마나 걸리나요?**
A: 일반적으로 2-5분이지만 시장 상황에 따라 다를 수 있습니다.

**Q: CoW Swap 주문이 만료되면 어떻게 되나요?**
A: 아무것도 잃지 않습니다. 토큰은 지갑에 남아 있습니다. 조정된 매개변수로 새 주문을 할 수 있습니다.

**Q: 프로토콜 경로가 때때로 사용할 수 없는 이유는 무엇인가요?**
A: **스테이킹의 경우:** 예치 풀이 가득 찼습니다. **언스테이킹의 경우:** ETH 유동성이 부족합니다. 이들은 사용자가 프로토콜과 상호 작용함에 따라 변경되는 임시 상태입니다.

**Q: 스테이킹 거래는 과세 대상인가요?**
A: 대부분의 관할 구역에서 그렇습니다. ETH를 rETH로 변환하고 되돌리는 것은 일반적으로 과세 사건입니다. 특정 상황에 대해서는 세무 전문가와 상담하세요.

**Q: rETH 취득 원가를 어떻게 추적하나요?**
A: ETH를 스테이킹할 때의 가치를 기록하세요 - 이것이 rETH 취득 원가가 됩니다. 정확한 기록을 위해 거래 해시, 타임스탬프 및 가격을 사용하세요.

**Q: 프로토콜과 CoW Swap 경로 간 이동은 세금에 차이가 있나요?**
A: 세금 처리는 일반적으로 동일합니다 - 둘 다 ETH를 rETH로 교환하는 것을 포함합니다. 선택한 경로는 일반적으로 세금 의무에 영향을 미치지 않지만 기록 보관은 약간 다를 수 있습니다.
