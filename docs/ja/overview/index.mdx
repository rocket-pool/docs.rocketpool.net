# イントロダクション

Rocket Pool は、Ethereum Proof of Stake プロトコルの先駆者であり、コミュニティ所有、分散型、信頼不要、Ethereum でのステーキングと互換性があるように設計されています。2016 年後半に最初に構想され、2021 年 10 月以来ライブになっています。

Rocket Pool は 2 つの主なユーザーグループに対応するように設計されています。[rETH を使用したトークン化ステーキング](https://medium.com/rocket-pool/rocket-pool-2-5-tokenised-staking-48601d52d924#92b0)に参加したい者で、わずか **0.01 ETH** で参加でき、ETH をステーキングしてネットワークでノードを実行し、取得したコミッションにより、プロトコル外でのステーキングよりも高い ROI を生成するのに役立つことを望む者です。

これら 2 つのグループがプロトコルを構成する方法の詳細については、[エクスプレーナーシリーズの記事 1 を読むことを強く推奨します](https://medium.com/rocket-pool/rocket-pool-staking-protocol-part-1-8be4859e5fbd)。このシリーズは、トークン化ステーキングの実行やノード実行のいずれであっても、ユーザーがプロトコルに参加する方法について詳しく説明しています。

プロトコルの基本的な前提は、ネットワークが 1 つの当事者に支配されないようにすることです。これは Ethereum と ETH 自体に直接関連する原則であり、Rocket Pool が進化する過程での各段階で使用される考え方です。

Rocket Pool は Ethereum と DeFi の中核的な倫理、特にノンカストディアルで信頼不要な性質を具現化し、真の自己主権を繁栄させることを目指しています。

これが ETH ステーキング用のベースプロトコルレイヤーを作成することが特に重要である理由です。特に、ノードを実行する技術的スキルがない、または 32 ETH を所有する経済的能力がない大多数のプレイヤーです。
