### Proof of Stakeとは何ですか?

Proof of StakeはEthereumで使用されるコンセンサスプロトコルです。Proof of Work (PoW) プロトコルとは異なります。PoWについては聞いたことがあるかもしれません。BitcoinがPoWを使用しているためです。分散型ブロックチェーンでは、これらのコンセンサスプロトコルは、どのトランザクションが処理され、どの順序で処理されたかを全員が同じページで把握するのに役立ちます。これは検証として知られています。

EthereumにはBeacon Chainと呼ばれるProof of Stake (PoS) チェーンがあり、PoWよりも高速で、エネルギー効率が高く、より分散化されています。ユーザーがETHを預金し、必要な検証を実行するために24時間365日オンラインでEthereumノードを提供することで動作します。ノードを提供することへの報酬として、Beacon ChainはNode Operatorに預金の上に追加のETHを与えます。これは、銀行口座で利息を得るのと同様に見ることができますが、これらの報酬はネットワークの保護を支援することの見返りとして生成されます。

### Rocket Poolは何をしますか?

Rocket Poolは、分散型でトラストレスなEthereumステーキングのための基盤層プロトコルです。

あらゆる規模のステーカーをサポートするように設計されたRocket Poolは、RPL担保（良好な行動を保証するボンディングまたは保険の一種）に支えられた完全な自律性を持つ分散型Node Operatorのネットワークに、誰でもトラストレスに最低**0.01** ETHをステークできるようにする意図で構築されました。**ETH**を**rETH**と交換することでステークできます。これは[分散型Node Operatorのネットワーク全体のパフォーマンスに基づいて時間とともにステーキング報酬を得るトークン](https://medium.com/rocket-pool/rocket-pool-2-5-tokenised-staking-48601d52d924#92b0)です。また、プロトコル内のNode Operatorとして**8または16 ETH**をステークし、自分のステークに対して手数料なしで報酬を得るとともに、ネットワークから手数料とRPL報酬を獲得し、ソロのNode Operatorとして外部でステーキングするよりもプロトコル内でステーキングすることでより高いROIを生み出すことができます。

### なぜ使用する必要があるのですか?

Rocket Poolは、個人、グループ、企業のためにEthereumのProof of Stakeに存在するいくつかの高い参入障壁を取り除きます。

- Beacon Chainは最低**32 ETH**の預金を必要とします。Rocket Poolでは、誰でも**0.01 ETH**という少額で報酬を得ることができます。

- Beacon ChainへのETHの預金には、Smart Contractとの対話方法についての知識も必要です。Rocket Poolは、Beacon Chainとのすべての対話を処理します。

- Rocket Poolで預金すると、すぐにrETHトークンを受け取ります。これは、時間とともに報酬を得るトークン化されたステーキング預金であり、私たちとロックする必要はありません。いつでも取引/販売/保有でき、ユーザーに流動性を提供します。また、流動性が利用可能であれば、いつでもRocket PoolにETH+報酬と交換することもできます。

- Beacon Chainは、預金をするユーザーが24時間365日Ethereumノードを実行し、そのノードをオンラインで安全に保つ技術的な能力を持っていることを要求します。Rocket Poolには、このサービスを提供する独立したEthereum Node Operatorの分散型ネットワークがあります。

- Beacon Chainは、預金をしてもノードをオンラインに保てないユーザーにペナルティを課します。Rocket Poolは、ノードで発生するペナルティや損失をネットワーク全体で社会化し、単一のユーザーへの影響を大幅に最小化します。

- 一部のユーザーは、自分のノードを実行することに自信があるかもしれませんが、預金に必要な32 ETHを持っていません。Rocket Poolの分散型Ethereum Node Operatorネットワークに参加すれば、ソロステークに最低8 ETHのみが必要です。また、このサービスを提供することでネットワークから追加収入を得られるという利点もあり、ソロでステーキングするよりも高いROIを生み出します。

### どのように機能しますか?

しっかりつかまってください。これは重要なものです!プロトコルは主に3つの主要要素で構成されています。Smart Contract、Smart Node Network、Minipool Validatorです。

- 私たちのSmart ContractはETHの預金を受け入れ、現在のNode Operator需要に基づくステーキング手数料率でNode Operatorに割り当て、さまざまなトークンを発行および追跡します。

- 私たちのSmart Node Networkは、Smart Nodeソフトウェアを実行する特別なEthereumノードの分散型ネットワークです。プロトコルのSmart Contractと通信できるカスタムバックグラウンドプロセスを備えており、同様に重要なことに、Beacon Chainが必要とするネットワークコンセンサス（検証）を提供します。

- どのユーザーも、最低8 ETHがあれば、これらのSmart Nodeの1つを実行し、自分のETHを手数料なしでステークできます。プロトコルにSmart Nodeを提供することで、ユーザーは自分のETHをステーキングして得る報酬の上に、ネットワークから追加の報酬も受け取ります。悪い行動や低いパフォーマンスに対する保険やボンドとして最初にRPLを担保として提供することで、RPLトークンを獲得します。

- Minipool Validatorは別のSmart Contractですが、プロトコルの重要な側面であるため、独自に定義する価値があります。これらのSmart Contractは、ノードに8または16 ETHを預金するNode Operatorによって作成されます。Node Operatorが8 ETHを預金する場合を考えてみましょう。彼らがこれを行うと、このSmart Contractは、ノードを実行せずにステークしたいだけのユーザー（rETHステーカー）からの預金として24 ETHを受け取ります。このContractに合計32 ETH（8 ETHの初期預金+ rETHステーカーからの24 ETH）が含まれると、その預金がステーキング報酬を得るためのコンセンサス義務を実行するノード上に新しいvalidatorが作成されます。素晴らしい!

### 独立したNode Operator vs Staking as a Service (SaaS) プロバイダー?

Rocket Poolプロトコルは、独立したNode Operatorとサービスプロバイダーを含む幅広いアクターをサポートしています。

Web3は、Ethereumのようなプロジェクトでproof of stakeの景観へのアクセスをより良くするのを助ける、非常に知識豊富なStaking as a Serviceプロバイダーでいっぱいです。彼らは、機関投資家からヘッジファンド、ファミリーオフィス、そしてその間のすべてをサポートしています。

Rocket Poolは、これらのプロバイダーをサポートするように設計されました。つまり、SaaSソリューションを通じてステークされたETHは、各クライアントに対処するためのカスタムステーキングソリューションをスピンアップする必要なく、Rocket Poolを通じて使用できます。

ETH保有者は、SaaSプロバイダーに支払うか、自分でオペレーターとして支払われるかを選択できます。Rocket Poolを使用すると、SaaSプロバイダーと独立したオペレーターは、ETHとRPLの両方でノードを実行することで支払われることで、リターンを最大化します。プロトコルにより、チームは独自のインフラストラクチャを実行し、Rocket Poolを使用して8または16 ETHのバッチでETHをトラストレスにステークできます—資本をさらに活用し、より大きなリターンシェアを獲得します。

この設計は、Bison TrailsやGeminiがDeFiのパワーユーザーと同じようにRocket Poolを使用できることを意味します。8または16 ETHを持って現れるだけで、他のNode Operatorと同じように扱われます。Rocket Poolの民主化されたステーキングシステムは、いかなる当事者も優遇しません。Rocket Poolを通じてステークされたETHは常にネットワークを直接サポートします。

Rocket PoolのステークされたETHラッパーであるrETHは、Rocket Poolネットワークが完全に分散され、信頼を最小化されているため、DeFiで最も純粋です。これにより、rETHはEthereumとあらゆる種類のプロバイダーにとって自然な構成要素となります。

### Rocket Poolはどのトークンを使用しますか?

**RPL — Rocket Pool Protocol Token**

RPLは、プロトコルのガバナンスで使用される主要なプロトコルトークンであり、Rocket Poolノードにステークすることもできます。

Node Operatorの観点から:

- Saturn 1の時点で、投票資格のあるステークされたRPLは、プロトコルの総収益のシェアを取得します
- RPLインフレーションの[大部分](https://rpips.rocketpool.net/RPIPs/RPIP-25)は、ステークされたRPLに基づいてNode Operatorへの報酬として与えられます。これらの報酬はSaturn 2で段階的に廃止される予定です
- 2024年10月28日以降に作成されたSaturn 0ミニプールの場合:ステークされたRPLは最大4%の追加ボーナス手数料を提供します。ボーナス手数料はSaturn 1の発売の数か月後に段階的に廃止される予定です
- ステークされたRPLはガバナンスのための投票権を提供します

プロトコルの観点から:

- RPLインフレーションの[少数](https://rpips.rocketpool.net/RPIPs/RPIP-25)は、pDAOへの資金を提供します
- RPLインフレーションの[少数](https://rpips.rocketpool.net/RPIPs/RPIP-25)は、oDAOへの資金を提供します
- 二次担保として機能できます（例:スラッシングに対して）

**rETH — Rocket Pool Staking Deposit Token**
ユーザーがプロトコルに預金すると、すぐにrETHトークンを受け取ります。これは、トークン化されたステーキング預金と、Rocket Poolネットワークで時間とともに得られる報酬を表します。

このトークンはネットワーク内でロックする必要はなく、ユーザーが望むように取引、販売、または保有できます。

このトークンは、流動性がプロトコルに十分ある場合、いつでもプロトコルにETH+獲得した報酬と交換することもできます。

### ステーキング期間はありますか?

ステークしたい場合は、預金時にrETHトークンを受け取ります。その後、このトークンを好きなようにできます。時間とともにステーキング報酬を得ることができ、売却/取引したり、取引に流動性が利用可能であればRocket PoolにETH+報酬と交換したりできます。

ネットワークでノードを実行したい場合は、希望する期間だけNode Operatorとしてステークできます。もちろん、長くステークすればするほど、より多くの報酬を得られます。

### Rocket Poolでステーキングする際の手数料はどうなりますか?

Node Operatorの手数料は、非オペレーターステーカーからプールされた16または24 ETHから生成された報酬から得られます。手数料は14%に固定されています。

### プロトコルはどのようにrETHの価値を保護しますか?

私たちのSmart Contractは、資金の保持と移動を担当しています。透明性のために、すべての人が分析できるように[オープンソース](https://github.com/rocket-pool/rocketpool)になっています。さらに、Smart Contractがメインネットに展開される前に、最高の監査チームの2つであるSigma PrimeとConsenSys Diligenceによって完全に監査されました。

Rocket Poolは、Node Operatorの分散型ネットワークです。非オペレーターステーカーの場合、預金はProof of Stake義務を実行するNode Operatorに割り当てられます。Node Operatorは、割り当てられたのと同じ量のETHをステークする必要があります。ノードのパフォーマンスが悪いためにETHの損失がある場合、オペレーターはまず元の8または16 ETHとRPLの損失でプールの失われたETHを補償しなければなりません。これが損失を完全に補償しない場合、残りは、悪いノードに預金が割り当てられただけで運が悪いために1人のユーザーが大きな損失を被るのではなく、rETHの相対価値の減少を通じてネットワーク全体に分散されます。

例えば、Node Operatorが16 ETHをステークし、そのノードが28 ETHでネットワークを離れた場合、オペレーターは12 ETHを保持し、ネットワークは16 ETHを保持します—すべての損失はオペレーターにあります。ノードが15 ETHで離れた場合、ネットワークは16 ETHを保持し、オペレーターは不足している1 ETHをRPLの損失を通じて補います。ノードが10 ETHで離れ、元のボンドから1.6 ETH相当のRPLしかない場合、ネットワークは11.6 ETH（10 + 1.6）を保持し、4.4 ETH（16 - 11.6）の損失はネットワーク全体に分散されます。

したがって、Node Operatorは良好なパフォーマンスを発揮する大きなインセンティブを持っています。また、Node Operatorは、パフォーマンスが悪い場合により大きな安全網を提供することで与えられる追加の報酬により、可能な限り多くのRPLを保険としてステークするようプロトコルによってインセンティブが与えられます。

### 分散型ステーキングプロトコルとは正確には何ですか?

Rocket Poolは、Node Operatorの分散型ネットワークを含むプロトコルです。各Node Operatorは、独自のサーバーインフラストラクチャを持つ別個のエンティティです。この異種ネットワークを持つことは、次の点で重要です:

- _耐障害性_ — 障害は少数のノードにのみ影響します
- _スケーラビリティ_ — ネットワークを開くことで、需要を満たすために膨大な数のノードをオンラインにできます
- _Quadratic Leaking_ — これは、Beacon Chainがオフラインのvalidatorをどのように罰するかを表す洗練された用語です。Node Operatorとして、他のvalidatorの大きなグループとともにオフラインになると、罰則はより大きくなります—これは中央集権的なステーキングサービスを抑止します。

### 預金と報酬にはどのようにアクセスできますか?

非オペレーターステーカーで、Rocket PoolのSmart Contractに預金を送信するだけの場合、すぐにrETHトークンを受け取ります。これはトークン化されたステーキングの一形態であり、時間とともに報酬を得ることができますが、私たちと一緒に保持する必要はなく、取引/販売したり、希望どおりに保持したりできます。また、交換をカバーする流動性がプロトコルに利用可能であれば、プロトコルにETH+獲得した報酬と交換することもできます。

### ソロで行うのではなく、Rocket Pool Smart Nodeを実行する理由は?

Node Operatorとして、次のような利点を活用できます:

- 素晴らしい[Smart Node Stack](https://github.com/rocket-pool/smartnode)を使用できます。これは、ステーキングを非常に簡単にする非常に便利なセットアップです。[こちらのインストーラーから始めましょう](https://github.com/rocket-pool/smartnode)。
- Rocket Poolがプロトコルの預金の24 ETHを割り当てるため、ステークするのに8 ETHのみが必要です（ソロでは32 ETHとは対照的に）。また、1つのノードで複数回8 ETHの預金をステークすることもできます。
- ノード上の彼らのETHによって獲得された報酬の14%の手数料をRocket Poolユーザーに請求することで、追加の報酬を得ることができます。
- 自分のETHをステークし、手数料から解放されます。
- ノードに担保としてRPLをステークし、この保険の約束を提供することでプロトコルからより多くのRPLを獲得します。
- 常に自分のノードを制御しています。
- いつでもネットワークから出入りできます。

### Node Operatorの最低ステーキング額はいくらですか?

Node Operatorの最低ステーキング額は8 ETHです。しかし、Smart Nodeソフトウェアを使用して、自分のノードで簡単に8 ETHの倍数でステークできます。また、深刻なダウンタイムやペナルティの場合の担保として、いくらかのRPLも必要になります。この担保は、ステークする期間が長いほどより多くのRPLを獲得するプロトコルへの保険の約束として扱われます。

### Rocket Poolノードの実行には何が含まれますか?

Node Operatorとして、proof of stakeのvalidator義務を実行する24時間365日のサーバーを実行する必要があります。プロトコル内のノード+validatorを管理するために、私たちの[Smart Node Stack](https://github.com/rocket-pool/smartnode)を使用してください。これは、ステーキングを非常に簡単にする非常に便利なパッケージです。[こちらのインストーラーから始めましょう](https://github.com/rocket-pool/smartnode)。

[Saturn 0アップグレード](/ja/upgrades/saturn-0/whats-new)以降、Rocket Poolプロトコルは最低限のRPL量を必要としなくなりました。[Saturn 0アップグレード](/ja/upgrades/saturn-0/whats-new)前は、プロトコルによって提供されたETHのRPL価値の10%の最低値（それぞれ24または16 ETHの10%、つまり2.4または1.6 ETH）が必要でした。

### 統合されたdApps、SaaSビジネスなど

dAppの場合、Rocket PoolのSmart Contractに直接統合して、dAppが保持する可能性のあるETHをステークできます。預金時にすぐにrETH（トークン化されたステーキング預金）を受け取り、dAppが望むように使用できます。これで、分散型ステーキングネットワークを使用した分散型dAppでステーキングができます。すごい!

SaaS（Staking as a Service）プロバイダーの場合、ユーザーのためにリターンを最大化したいかもしれません。Rocket Poolネットワークでいくつかのスマートノードをスピンアップし、そのETHをステーキング報酬+追加手数料+ RPL報酬を獲得するために機能させます。

ウォレットまたはDEXで、ユーザーがサービスを離れることなくETHをステークできるようにしたいですか?ユーザーに代わってRocket PoolにETHを預金できるSmart Contractを書き、すぐにrETHを受け取ってください。完了!ステーキングサービスを提供しました。
さらに多くの可能性があります。Rocket Poolは、許可不要のSmart Contractベースのステーキングプロトコルであり、誰でもプラグインできます。ユーザーは、私たちを使用していることを知る必要さえありません。それは完全にあなた次第です!
