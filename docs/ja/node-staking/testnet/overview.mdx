# テストネットワークでの練習

イーサリアムの**テストネットワーク**（略して `testnet`）は、イーサリアムブロックチェーンと機能的に同一ですが、スマートコントラクトをメインの「本物の」ブロックチェーン（`mainnet` と呼ばれる）にライブ展開する前にテストするために使用されることを目的としています。
テストネットはメインネットと並行して実行されています - 実際には現在いくつかのテストネットが実行されています！

興味があれば、いくつかの実行レイヤーテストネットについて[ここ](https://support.mycrypto.com/how-to/getting-started/where-to-get-testnet-ether)で詳しく読むことができます。
Rocket Pool は実行レイヤーテストネットとして **Hoodi** を使用しています。
こちらでブロックエクスプローラーを使用して確認できます: [https://hoodi.etherscan.io/](https://hoodi.etherscan.io/)

同じ概念がコンセンサスレイヤーにも適用されます。
Rocket Pool はコンセンサスレイヤーテストネットとして **Hoodi** を使用しています。
こちらでブロックエクスプローラーを見つけることができます: [https://hoodi.beaconcha.in/](https://hoodi.beaconcha.in/)

ネットワークはテスト用であるため、**ネットワーク上の ETH は人々のテストを支援するために無料で配布されています**。
この ETH はメインネットに**転送できません**。Hoodi に留まり、実際の価値はないため、**モノポリーマネー**と考えることができます。

::: danger
**Rocket Pool のテストネットワークを試す場合、いかなる時点でもメインネットの実際の ETH を使用しないでください！**
これを行うと、**資金を永久に失います**！
フォーセットから Hoodi ネットワークで受け取ったテスト ETH のみを使用する必要があります（以下で説明します）！
:::

## テストネットの構成

[Docker を使用した Rocket Pool のデフォルト構成](../install-modes)は、テストネットの使用に関連するすべての設定を自動的に処理します。
特別なことをする必要はありません。

外部実行クライアントまたはコンセンサスクライアントを使用した[異なるセットアップ](../install-modes)を使用している場合、または[Docker なしでネイティブに](../install-modes)クライアントを実行している場合は、これを手動で設定する必要があります:

- **Geth** の場合: 起動コマンドに `--hoodi` を追加します。
- **Lighthouse** の場合: 起動コマンドに `--network hoodi` を追加します。
- **Nimbus** の場合: 起動コマンドに `--network=hoodi` を追加します。
- **Prysm** の場合: 起動コマンドに `--hoodi` を追加します。
- **Teku** の場合: 起動コマンドに `--network=hoodi` を追加します。
- **Lodestar** の場合: 起動コマンドの前に `LODESTAR_NETWORK=hoodi` を付けます。

次のセクションをチェックして、Rocket Pool をテストするために必要な Hoodi でのテスト ETH の取得方法を学んでください。

## Hoodi でのテスト ETH の取得

Rocket Pool テストネットワークを使用するには、Hoodi で ETH が必要です。
幸いなことに、**フォーセット**（テスト用の Hoodi ETH を提供する自動化サービス）を使用して無料で入手できます。

::: warning 注意
テスト ETH を受け取るには、イーサリアムアドレスを提供する必要があります。

ETH を通常ステーキングして rETH を受け取ろうとしているだけの場合は、Hoodi ETH を取得する前に[ステーキングガイド](../../liquid-staking/overview)を確認してください。
適切な手順で戻ってくるよう促されます。

ノードを実行する予定がある場合は、まず[ノードオペレーターガイド](../responsibilities)に従ってノードをセットアップし、新しいウォレットを作成してください。
適切な手順で戻ってくるよう促されます。
:::

Rocket Pool は [Discord サーバー](https://discord.gg/rocketpool)でフォーセットを提供しています。
サーバーに参加して `#support` チャンネルに向かいます。

ここで、テストネット ETH を要求できます。
ノードアドレスを提供する必要があります。

::: warning 注意
テストネットフォーセットボットは、以前の悪用のため一般に直接アクセスできなくなりました。
開発チームとコミュニティの著名なメンバーのみが利用できます。
テストネット eth を受け取るには、以下のコマンドを実行し、チャンネルで意図を述べてください。
辛抱強く待てば、誰かがあなたに代わってフォーセットにアクセスします。
:::

ノードを登録する予定がなく、rETH を受け取るためにステーキングをテストするための ETH だけが欲しい場合は、チャンネルで次のコマンドを入力します:

```
!hoodiEth <your hoodi address>
```

山括弧は付けません。
ウォレットに ETH がまだない場合、ボットはテスト用に 1 ETH をウォレットに送信します。

ノードを実行する予定がある場合は、次の手順に従います:

1. まず上記の `!hoodiEth` コマンドを実行して、ノードを登録するのに十分な ETH を取得します
2. ノードを登録します（手順については[ノードの準備](../prepare-node)ページを参照してください）
3. 次のコマンドを実行して 8 Hoodi ETH を取得し、バリデーターを作成できるようにします:

```
!hoodiOperator <your hoodi address>
```

`!hoodiOperator` を使用するには、提供するアドレスが登録された Rocket Pool ノードである必要があります。
したがって、この順序でこれらの 3 つの手順に従う必要があります。

## Hoodi でのテスト RPL の取得

ノード運用をテストしている場合、テスト ETH に加えてテスト RPL が必要です。

Rocket Pool は [Discord サーバー](https://discord.gg/rocketpool)でフォーセットを提供しています。
サーバーに参加して `#support` チャンネルに向かいます。

ここで、テストネット RPL を要求できます。
ノードアドレスを提供する必要があります。

::: warning 注意
テストネットフォーセットボットは、以前の悪用のため一般に直接アクセスできなくなりました。
開発チームとコミュニティの著名なメンバーのみが利用できます。
テストネット RPL を受け取るには、以下のコマンドを実行し、チャンネルで意図を述べてください。
辛抱強く待てば、誰かがあなたに代わってフォーセットにアクセスします。
:::

```
!hoodiRpl <your hoodi node address>
```

これにより、メインネットのオリジナル RPL トークンに類似した **レガシー（v1）RPL** トークンが 600 提供されます。
Rocket Pool は、ノードオペレーターに報酬を与えるために使用されるインフレーションをサポートする**新しい（v2）RPL** トークンを使用しています。

レガシー RPL を新しい RPL にスワップする方法については、[ミニプール（バリデーター）の作成](../create-validator)ガイドの手順に従ってください。
