# 以前のベータテストからSmartnodeを移行する

MedallaやPyrmontでの以前のベータテストでRocket Poolのnode operatorをされていた方は、プロトコルの改善にご協力いただきありがとうございます。そして、おかえりなさい!
この簡単なガイドでは、以前のベータビルドをパージして、Hoodiテストネット用、またはメインネットでの実際のステーキング用の最新ビルドをセットアップするプロセスについて説明します。

      ベータビルドからHoodiテストネットワークへのアップグレードには、3つの主要なステップがあります。

      まず、お持ちのhoodi ETHを別のウォレットに移行します。
      Rocket Poolを再インストールした後に作成する新しいノードウォレットに移行できるため、hoodi ETHのフォーセットにあまり依存する必要はありません。

      最も簡単な方法はMetamask経由です - 新しいウォレットのセットアップ方法については、[rETHステーキングガイド](../liquid-staking/overview)をご覧ください。
      次のコマンドを使用してこれを行うことができます。

      ```shell
      rocketpool node send XXX eth 0xabcd1234...
      ```

      ここで、`XXX`は送信するhoodi ETHの量で、`0xabcd1234...`は作成した新しいウォレットのアドレスです。
      **トランザクションで使用されるガスを支払うために、少量のETHを残しておく必要があることに注意してください!**

      次に、次のコマンドですべての古いDockerイメージとチェーンデータを削除します。

      ```shell
      rocketpool service terminate
      ```

      これが完了したら、古いRocket Poolのデータと設定フォルダを削除します。
      **これにより、古いウォレットと古いバリデーターが削除されるため、もう何も保存する必要がないことを確認してください!**

      ```shell
      sudo rm -rf ~/.rocketpool
      ```

      これが完了すると、古いベータセットアップがクリーンアップされます。
      Hoodiテストネット用のRocket Poolのインストールと設定のウォークスルーについては、[Rocket Poolインストールガイド](../node-staking/eth-clients)に従ってください。
