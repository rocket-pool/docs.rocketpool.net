# Миграция Smartnode из предыдущих бета-тестов

Если вы были оператором узла Rocket Pool во время наших предыдущих бета-тестов на Medalla или Pyrmont, спасибо за помощь в улучшении протокола и добро пожаловать обратно!
Это краткое руководство проведет вас через процесс удаления вашей бета-сборки и настройки новейшей сборки либо для тестовой сети Hoodi, либо для реального стейкинга в основной сети.

      Существует три основных шага к обновлению с бета-сборки до тестовой сети Hoodi.

      Сначала переведите любые hoodi ETH, которые у вас есть, в отдельный кошелек.
      Затем вы сможете перевести их обратно в новый кошелек узла, который вы создадите после переустановки Rocket Pool, чтобы не зависеть от фаусета hoodi ETH.

      Самый простой способ сделать это - через Metamask - см. [руководство по стейкингу rETH](../liquid-staking/overview) для пошаговой инструкции по настройке нового кошелька.
      Вы можете сделать это с помощью следующей команды:

      ```shell
      rocketpool node send XXX eth 0xabcd1234...
      ```

      где `XXX` - это количество hoodi ETH для отправки, а `0xabcd1234...` - это адрес нового кошелька, который вы создали.
      **Обратите внимание, что вам нужно оставить немного ETH для оплаты газа, используемого в транзакции!**

      Затем удалите все старые образы Docker и данные цепи с помощью следующей команды:

      ```shell
      rocketpool service terminate
      ```

      После этого удалите старую папку данных и настроек Rocket Pool.
      **Это удалит ваш старый кошелек и ваши старые валидаторы, поэтому убедитесь, что они вам больше не нужны!**

      ```shell
      sudo rm -rf ~/.rocketpool
      ```

      После этого ваша старая бета-настройка будет очищена.
      Вы можете следовать [руководству по установке Rocket Pool](../node-staking/eth-clients) для пошаговой инструкции по установке и настройке Rocket Pool для тестовой сети Hoodi.
