# Глоссарий

В документации Rocket Pool вы можете столкнуться с некоторыми незнакомыми концепциями. В этом разделе перечислены общие термины для удобного доступа, изучения и разработки плагинов/тем.

## APY - Годовая процентная доходность

Сумма прибыли (дохода), рассчитанная за один год и разделенная на первоначальную сумму инвестиций, выраженная в процентах.

## Залог (Bonding)

_См. Страхование._

## Контракт, (Смарт)

Программа Ethereum (смарт-контракт), которая используется для выполнения набора запрограммированных функций. Смарт-контракты Rocket Pool включают пул депозитов, формирование валидаторов minipools и выпуск и отслеживание различных взаимодействий с токенами в сети Ethereum.

## Клиент

Клиент - это реализация программного обеспечения Ethereum, которая проверяет все транзакции в каждом блоке, обеспечивая безопасность сети и точность данных. _См. также клиенты уровня исполнения и консенсуса._

## Кастодиальный

Централизованный сервис, который управляет всем процессом стейкинга ETH от имени пользователя и сохраняет "опеку" над приватными ключами валидатора и ключами вывода средств. _См. также некастодиальный._

## DAO (Децентрализованная автономная организация)

Организация, представленная правилами, закодированными в виде компьютерной программы, которая является прозрачной, контролируется членами организации и не зависит от центрального правительства.

## Deposit Pool

Количество ETH, готового к стейкингу. Пул депозитов пополняется обычными пользователями, когда они обменивают ETH на rETH. Этот ETH ожидает сопряжения с операторами нод в очереди операторов нод.

## ETH

Основная криптовалюта, генерируемая и используемая протоколом Ethereum.

## Execution Client

Программные клиенты Ethereum, которые запускают уровень исполнения.

## Consensus Client

Программные клиенты Ethereum, которые запускают уровень консенсуса.

## eth2.0 (Beaconchain)

Сеть Ethereum 2.0 и обновление сети Ethereum, которое было запущено в декабре 2020 года и заменило механизм консенсуса proof-of-work eth1.0 в сентябре 2022 года. Устаревший термин - теперь известен просто как Beaconchain или уровень консенсуса.

## Страхование (Insurance)

Количество токенов RPL, выраженное в процентах от стоимости ETH, предоставленного протоколом, которое оператор ноды должен внести при запуске валидатора minipool. Застейканный RPL оператора ноды будет выступать в качестве страховки для возмещения обычным пользователям в случае, если оператор ноды выйдет из стейкинга с меньшим количеством предоставленного протоколом ETH в своих валидаторах (24 ETH в случае minipool на 8 ETH, 16 ETH в случае minipool на 16 ETH).

## Minipool (Валидатор)

Валидатор, который был профинансирован и инициализирован через программное обеспечение Rocket Pool. Он состоит из ETH, внесенного обычными пользователями через пул депозитов, и ETH, внесенного операторами нод во время процесса регистрации.

## Node

В сети Ethereum ноды - это устройства (компьютеры), которые запускают клиентское программное обеспечение Ethereum.

## Node Commission

Количество дополнительного ETH, присуждаемого оператору ноды. Это фактически "комиссия", которая взимается с обычных стейкеров; в Rocket Pool эта комиссия передается операторам нод. Когда Rocket Pool был запущен, она составляла от 5 до 20% с идеальной целью в 10%, при этом комиссия каждого minipool фиксировалась на всю жизнь при создании minipool. Новые активированные minipools теперь имеют установленную ставку комиссии в 14%.

## Node Operator

Физическое лицо, которое хочет обеспечить безопасность сети Ethereum, управляя нодой (компьютером), которая будет запускать программное обеспечение Rocket Pool, клиент валидатора и, опционально, клиент исполнения и beaconchain. Операторы нод будут получать вознаграждения в виде ETH APY по более высокой ставке, чем соло-стейкеры в сети Ethereum.

## Node Software

Набор программ, которые позволяют операторам нод взаимодействовать с протоколом Rocket Pool.

## Node Wallet

Кошелек Ethereum, который генерируется Rocket Pool при создании новой ноды. Этот кошелек будет использоваться для финансирования новых minipools во время их создания и оплаты комиссий за газ, необходимых для взаимодействия со смарт-контрактами Rocket Pool.

## Некастодиальный (Non-Custodial)

Сервис, который обеспечивает упрощение настройки и управления валидаторами, но не хранит приватные ключи валидатора и ключи вывода средств пользователя. Rocket Pool - это некастодиальный сервис стейкинга.

## Oracle DAO (oDAO)

DAO, состоящая из всех операторов нод-оракулов. _См. также DAO._

## Oracle Node

Специальная нода, управляемая доверенными членами сообщества Rocket Pool, которая передает информацию с beacon chain (такую как общие вознаграждения за стейкинг и статус выхода валидатора) обратно на уровень исполнения, чтобы смарт-контракты Rocket Pool могли функционировать корректно.

## Proof-of-Stake (PoS)

Механизм консенсуса, используемый в Ethereum, где пользователи голосуют за корректность новых блоков, застейкав ETH в своем кошельке.

## Protocol DAO (pDAO)

DAO, состоящая из всех держателей токенов RPL. _См. также DAO._

## Regular Node (Operator)

Термин, присвоенный "обычным" операторам нод, которые не имеют дополнительных обязанностей Oracle Nodes.

## Regular Stakers

Розничный (обычный) пользователь, который хочет зарабатывать проценты на своих монетах ETH без отделения их от базовой стоимости ETH. Обычный стейкер взаимодействует с платформой Rocket Pool, обменивая монеты ETH на токены rETH. Их ETH попадает в пул стейкинга, где он приобретается операторами нод при создании новых minipools.

## rETH (Rocket Pool Staking Deposit Token)

Токен, который представляет депозит обычного пользователя в пуле стейкинга. Пользователи получают эквивалентную стоимость токена rETH за каждую монету ETH, которую они вносят. По мере получения процентов от стейкинга в сети Ethereum стоимость токена rETH увеличивается по отношению к ETH. Этот токен не нужно блокировать в сети, и его можно торговать, продавать или держать по желанию пользователя.

## RPL (Rocket Pool Token)

**RPL — Токен протокола Rocket Pool**

RPL - это основной токен протокола, который будет использоваться в управлении протоколом, а также может быть застейкан на ноде Rocket Pool.

С точки зрения оператора ноды:

- С версии Saturn 1 застейканный RPL с правом голоса будет получать долю от общего дохода протокола
- [Большинство](https://rpips.rocketpool.net/RPIPs/RPIP-25) инфляции RPL выдается в качестве вознаграждений операторам нод на основе застейканного RPL; эти вознаграждения планируется постепенно отменить с Saturn 2
- Для minipools Saturn 0 (созданных после 28.10.2024): Застейканный RPL обеспечивает до 4% дополнительной бонусной комиссии; бонусная комиссия планируется к постепенной отмене через несколько месяцев после запуска Saturn 1.
- Застейканный RPL предоставляет право голоса в управлении

С точки зрения протокола:

- [Меньшинство](https://rpips.rocketpool.net/RPIPs/RPIP-25) инфляции RPL обеспечивает финансирование для pDAO
- [Меньшинство](https://rpips.rocketpool.net/RPIPs/RPIP-25) инфляции RPL обеспечивает финансирование для oDAO
- Может служить вторичным обеспечением (например, против слэшинга)

## Stake

Количество монет или токенов, которое вносится в качестве залога для выполнения некоторой работы. Физическое лицо может застейкать ETH либо как обычный пользователь, обменивая ETH на rETH, либо как оператор ноды, внося ETH для использования в процессе консенсуса PoS Beaconchain. Лицо, которое стейкает свой ETH (например, стейкер), вознаграждается APY или доходом на свой внесенный ETH.

## Staking Pool

Пул стейкинга позволяет нескольким держателям монет ETH объединить свои токены для коллективного формирования необходимой суммы для запуска валидатора Ethereum. Rocket Pool формирует, отслеживает, назначает и страхует эти коллективные пулы, используя некастодиальный децентрализованный метод.
