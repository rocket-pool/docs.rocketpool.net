### 什么是权益证明？

权益证明是以太坊使用的共识协议。它与工作量证明 (PoW) 协议不同，你可能之前听说过 PoW，因为比特币使用的就是 PoW。在去中心化区块链中，这些共识协议帮助每个人保持同步，让我们都知道哪些交易已被处理以及处理顺序。这被称为验证。

以太坊有一条权益证明 (PoS) 链，称为信标链，它比 PoW 更快、更节能、更去中心化。它由存入 ETH 并提供 24/7 在线以太坊节点来执行所需验证的用户驱动。作为提供节点的奖励，信标链会在节点运营商的存款基础上给予额外的 ETH。这可以类比为在银行账户中赚取利息，不同之处在于这些奖励是为帮助保护网络安全而产生的回报。

### Rocket Pool 是做什么的？

Rocket Pool 是去中心化和无需信任的以太坊质押的基础层协议。

Rocket Pool 旨在支持各种类型和规模的质押者，允许任何人以最少 **0.01** ETH 无需信任地质押到由 RPL 抵押品支持的去中心化节点运营商网络中（RPL 抵押品是一种确保良好行为的保证金或保险）。你可以通过将 **ETH** 兑换为 **rETH** 来进行质押，rETH 是一种[随时间累积质押奖励的代币](https://medium.com/rocket-pool/rocket-pool-2-5-tokenised-staking-48601d52d924#92b0)，其收益基于整个去中心化节点运营商网络的表现。你也可以作为协议中的节点运营商质押 **8 或 16 ETH**，免费赚取自己质押的奖励，同时从网络中赚取佣金和 RPL 奖励，与作为独立节点运营商在协议外质押相比，在协议内质押能获得更高的投资回报率。

### 为什么我需要使用它？

Rocket Pool 消除了以太坊权益证明对个人、团体和企业存在的多个高准入门槛。

- 信标链要求至少存入 **32 ETH**。Rocket Pool 允许任何人以最少 **0.01 ETH** 的存款赚取奖励。

- 在信标链上存入 ETH 还需要用户了解如何与智能合约交互。Rocket Pool 为你处理与信标链的所有交互。

- 使用 Rocket Pool，你在存款时会立即获得我们的 rETH 代币。这是一种代币化的质押存款，随时间累积奖励，无需锁定在我们这里。它可以随时交易/出售/持有，为用户提供流动性。如果有可用流动性，它也可以随时在 Rocket Pool 兑换回 ETH 加奖励。

- 信标链要求存款用户具备 24/7 运行以太坊节点的技术能力，并保持该节点在线和安全。Rocket Pool 拥有提供此服务的去中心化独立以太坊节点运营商网络。

- 信标链会惩罚存款但未能保持节点在线的用户。Rocket Pool 将节点上发生的任何惩罚或损失分摊到整个网络，大大减少了对任何单个用户的影响。

- 一些用户可能有信心运行自己的节点，但没有存款所需的 32 ETH。如果他们加入 Rocket Pool 的去中心化以太坊节点运营商网络，他们只需要最少 8 ETH 即可独立质押。他们还能获得额外收益，因为提供此服务可从网络获得额外收入，比独立质押产生更高的投资回报率。

### 它是如何运作的？

系好安全带，这是个重要内容！协议主要由 3 个主要元素组成：智能合约、智能节点网络和 Minipool 验证器。

- 我们的智能合约接受 ETH 存款，根据当前节点运营商需求将其分配给具有质押佣金率的节点运营商，并发行和跟踪各种代币。

- 我们的智能节点网络是运行我们智能节点软件的特殊以太坊节点的去中心化网络。它们具有自定义后台进程，允许它们与协议的智能合约通信，同样重要的是，提供信标链所需的网络共识（验证）。

- 任何用户都可以运行这些智能节点之一，如果他们有最低要求的 8 ETH，可以免费质押自己的 ETH。通过为协议提供智能节点，用户除了质押自己 ETH 获得的奖励外，还会从网络获得额外奖励。他们通过最初抵押 RPL 作为抵押品来获得 RPL 代币，为其节点提供针对不良行为或低性能的保险或保证金。

- Minipool 验证器是另一种智能合约，但由于是协议的关键方面，值得单独定义。这些智能合约由在其节点上存入 8 或 16 ETH 的节点运营商创建。让我们考虑节点运营商选择存入 8 ETH 的情况。当他们这样做时，这个智能合约会收到来自只想质押但不运行节点的用户（rETH 质押者）的 24 ETH 存款。当这个合约总共包含 32 ETH（8 ETH 初始存款加上来自 rETH 质押者的 24 ETH）时，会在节点上创建一个新的验证器，为该存款执行共识职责以赚取质押奖励。很棒吧！

### 独立节点运营商与质押即服务 (SaaS) 提供商？

Rocket Pool 协议支持广泛的参与者，包括独立节点运营商和服务提供商。

Web3 充满了知识渊博的质押即服务提供商，帮助世界更好地通过以太坊等项目进入权益证明领域。他们支持从机构资本到对冲基金、家族办公室等各种客户。

Rocket Pool 旨在支持这些提供商，这意味着通过 SaaS 解决方案质押的 ETH 可以通过 Rocket Pool 使用，而不必为每个客户建立定制的质押解决方案。

ETH 持有者可以选择付费给 SaaS 提供商，或者自己成为运营商获得报酬。通过 Rocket Pool，SaaS 提供商和独立运营商可以通过运行节点获得 ETH 和 RPL 双重报酬来最大化回报。该协议允许团队运行自己的基础设施，并使用 Rocket Pool 以 8 或 16 ETH 为单位无需信任地质押 ETH——让他们的资本发挥更大作用并获得更大份额的回报。

这种设计意味着 Bison Trails 或 Gemini 可以像 DeFi 高级用户一样使用 Rocket Pool。只需带着 8 或 16 ETH 出现，你就会被视为与任何其他节点运营商一样。Rocket Pool 的民主化质押系统不偏向任何一方，因为通过 Rocket Pool 质押的 ETH 始终直接支持网络。

Rocket Pool 的质押 ETH 包装代币 rETH 是 DeFi 中最纯粹的，因为 Rocket Pool 网络是完全分布式和信任最小化的。这使得 rETH 成为以太坊和各种提供商的自然构建块。

### Rocket Pool 使用什么代币？

**RPL — Rocket Pool 协议代币**

RPL 是将用于协议治理的主要协议代币，也可以在 Rocket Pool 节点上质押。

从节点运营商的角度来看：

- 从 Saturn 1 开始，符合投票条件的质押 RPL 将获得总协议收入的份额
- RPL 通胀的[大部分](https://rpips.rocketpool.net/RPIPs/RPIP-25)作为奖励根据质押的 RPL 分配给节点运营商；这些奖励计划在 Saturn 2 中逐步取消
- 对于 2024 年 10 月 28 日之后创建的 Saturn 0 minipool：质押的 RPL 可提供高达 4% 的额外奖励佣金；奖励佣金计划在 Saturn 1 启动几个月后逐步取消
- 质押提供治理投票权

从协议角度来看：

- RPL 通胀的[少部分](https://rpips.rocketpool.net/RPIPs/RPIP-25)为 pDAO 提供资金
- RPL 通胀的[少部分](https://rpips.rocketpool.net/RPIPs/RPIP-25)为 oDAO 提供资金
- 它可以作为二级抵押品（例如，针对罚没）

**rETH — Rocket Pool 质押存款代币**
当用户存入协议时，他们将立即收到 rETH 代币，该代币代表代币化的质押存款及其在 Rocket Pool 网络中随时间获得的奖励。

此代币无需锁定在网络中，可以根据用户意愿进行交易、出售或持有。

如果协议中有足够的流动性来满足交易，此代币也可以随时在协议中兑换回 ETH + 赚取的奖励。

### 协议如何保护 rETH 的价值？

我们的智能合约负责持有和转移资金。为了透明起见，它们是[开源的](https://github.com/rocket-pool/rocketpool)，供所有人分析。此外，在智能合约部署到主网之前，它们已经过 Sigma Prime 和 ConsenSys Diligence 两个顶级审计团队的全面审计。

Rocket Pool 是一个去中心化的节点运营商网络。如果你是非运营商质押者，你的存款将被分配给一个执行权益证明职责的节点运营商。节点运营商需要质押与分配给他们的 ETH 相等的金额。如果由于节点性能不佳而造成 ETH 损失，运营商必须首先用他们原始的 8 或 16 ETH 和 RPL 损失来补偿池子损失的 ETH。如果这不能完全弥补损失，其余部分将通过降低 rETH 的相对价值在整个网络中分摊，而不是让一个用户仅仅因为运气不好（他们的存款碰巧在一个不好的节点上）而承受巨大损失。

例如，如果节点运营商质押了 16 ETH，他们的节点以 28 ETH 离开网络，运营商保留 12 ETH，网络保留 16 ETH——所有损失都由运营商承担。如果他们的节点以 15 ETH 离开，网络保留 16 ETH，运营商通过损失 RPL 来弥补缺失的 1 ETH。如果节点以 10 ETH 离开，而原始保证金中只有价值 1.6 ETH 的 RPL，网络保留 11.6 ETH（10 + 1.6），4.4 ETH（16 - 11.6）的损失在整个网络中分摊。

因此，节点运营商有很大的动力表现良好。协议还激励节点运营商尽可能多地质押 RPL 作为保险，因为如果他们表现不佳，提供更大的安全网可以获得额外奖励。

### 什么是去中心化质押协议？

Rocket Pool 是一个包含去中心化节点运营商网络的协议。每个节点运营商都是拥有自己服务器基础设施的独立实体。拥有这种异构网络很重要，原因如下：

- _容错性_ — 故障只影响少数节点
- _可扩展性_ — 通过开放网络，意味着可以上线大量节点来满足需求
- _二次泄漏_ — 这是信标链惩罚离线验证者的专业术语。作为节点运营商，如果你与大量其他验证者一起离线，你的惩罚会更大——这阻止了中心化质押服务。

### 如何访问我的存款和奖励？

如果你是非运营商质押者，只是将存款发送到 Rocket Pool 的智能合约，你将立即收到 rETH 代币，这是一种代币化质押形式，它们仍会随时间累积奖励，但你不需要在我们这里持有它们，可以根据自己的意愿交易/出售或持有。如果协议中有流动性来支持兑换，你也可以将它们在协议中兑换回 ETH + 获得的奖励。

### 你们的智能合约是否经过审计并开源？

### 为什么我要运行 Rocket Pool 智能节点而不是独立质押？

作为节点运营商，你可以享受以下好处：

- 你可以使用我们出色的 [智能节点堆栈](https://github.com/rocket-pool/smartnode)，这是一个超级方便的设置，使质押变得非常简单。[从安装程序开始](https://github.com/rocket-pool/smartnode)。
- 你只需要 8 ETH 即可质押（而不是独立质押需要的 32 ETH），因为 Rocket Pool 会分配协议存款中的 24 ETH。你也可以在一个节点上多次质押 8 ETH 存款。
- 你通过向 Rocket Pool 用户收取他们 ETH 在你节点上赚取奖励的 14% 佣金来获得额外奖励。
- 你质押自己的 ETH，不收取任何佣金。
- 你在节点上质押 RPL 作为抵押品，并从协议中赚取更多 RPL 作为提供此保险承诺的回报。
- 你始终控制自己的节点。
- 你可以随时进出网络。

### 节点运营商的最低质押金额是多少？

节点运营商的最低质押金额是 8 ETH。但你可以使用我们的智能节点软件轻松地在自己的节点上以 8 ETH 的倍数进行质押。你还需要一些 RPL 作为抵押品，以防严重停机或处罚。这种抵押品被视为对协议的保险承诺，质押时间越长，你赚取的 RPL 越多。

### 运行 Rocket Pool 节点需要什么？

作为节点运营商，你需要运行一台 24/7 执行权益证明验证器职责的服务器。使用我们的[智能节点堆栈](https://github.com/rocket-pool/smartnode)来管理协议中的节点和验证器，这是一个超级方便的软件包，使质押变得非常简单。[从安装程序开始](https://github.com/rocket-pool/smartnode)。

自 [Saturn 0 升级](/zh/upgrades/saturn-0/whats-new) 以来，Rocket Pool 协议不再要求最低 RPL 金额。在 [Saturn 0 升级](/zh/upgrades/saturn-0/whats-new) 之前，需要协议提供的 ETH 价值的 10% 的 RPL（分别为 24 或 16 ETH 的 10%，即 2.4 或 1.6 ETH）。

### 集成 dApps、SaaS 业务等

如果你是 dApp，你可以直接集成到 Rocket Pool 的智能合约中，质押你的 dApp 可能持有的 ETH。存款时你将立即收到 rETH（代币化的质押存款），可以按你的 dApp 的意愿使用。你现在可以在一个使用去中心化质押网络的去中心化 dApp 中进行质押，太棒了！

如果你是 SaaS（质押即服务）提供商，你可能希望为用户最大化回报。在 Rocket Pool 网络中启动几个智能节点，让那些 ETH 发挥作用，赚取质押奖励 + 额外佣金 + RPL 奖励。

你是钱包或 DEX，想给用户提供质押 ETH 的能力而不离开你的服务吗？只需编写一个智能合约，可以代表用户将 ETH 存入 Rocket Pool，然后立即收到 rETH。砰！质押服务已交付。
还有更多可能性。Rocket Pool 是一个无需许可的基于智能合约的质押协议，任何人都可以接入。你的用户甚至不必知道你在使用我们，这完全取决于你！
