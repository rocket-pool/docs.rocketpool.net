# 代表节点质押 ETH

RPIP-32 允许帐户[代表](https://rpips.rocketpool.net/RPIPs/RPIP-32)在协议中注册的 Rocket Pool 节点质押 ETH。这支持节点运营者不直接提供 ETH 的各种情况：

- 增强节点运营者的安全性，因为他们可以直接从硬件钱包质押，无需事先将资金转移到节点。
- 质押即服务提供商，由受信任的托管人管理资金托管。
- 协议集成，由智能合约管理资金托管。
- DAO 或组织，由国库管理资金托管。

虽然此功能的主要目的是促进单个存款人的场景，但值得注意的是，多个独立存款人也可以通过在其上创建智能合约层来利用此功能。Rocket Pool 还在我们之前的 Atlas 版本中引入了[代表节点质押 RPL](../../node-staking/create-validator#whitelisting-an-address-to-stake-on-behalf) 的能力。

### 前提条件

确保您已正确配置您打算资助的节点的[主提取地址](../../node-staking/prepare-node#setting-your-primary-withdrawal-address)。点击上面的链接，了解有关 houston 升级后提取地址如何工作的指南。

::: tip 注意
不要跳过这一步！所有 ETH 保证金/奖励将提取到主提取地址。换句话说，为节点提供 ETH 的地址应该用作主提取地址！请确保节点的主提取地址设置为您拥有托管权的钱包。
:::

到达网站后，单击**连接钱包**按钮。请仔细阅读并接受服务条款和隐私政策，这将启用不同的连接方式，然后单击**连接 metamask**。

选择一个，确认一些权限。您可以通过单击窗口右上角的钱包图标来查看余额概览。

从这里，您需要输入要代表其存入 ETH 的节点的地址。页面应如下所示：

<img src="./images/enter_node_address.png" width="100%" height="auto" />

继续将您想要的地址粘贴到此框中，然后单击查找。在此示例中，我们将使用 `0xd3e317806456102d19db283b4c2201f0ef41a296`

<img src="./images/eth_on_behalf.png" width="100%" height="auto" />

在第一个框中，输入您想代表节点存入的 ETH 金额。第二个框指示您将代表哪个地址存入 ETH。请确保您输入了正确的地址和存款。一旦您满意，请单击存款，MetaMask 将弹出一个窗口要求您确认交易。

一旦您确认它，交易将开始。
当它被包含在一个区块中并添加到链中时，您的代表将能够使用 smartnode 中的 `rocketpool node status` 命令查看新余额。也可以通过 Rocket Pool 质押网站查看此余额。

```
The node has 0.000000 ETH in its credit balance and 8.000000 ETH staked on its behalf. 8.000000 can be used to make new minipools.
```

::: warning 注意
如果节点运营者将您的 ETH 存入 minipool，如果没有访问其提取凭证，您将无法退出他们的 minipool，因此请记住这一点！如果 ETH 未在 minipool 中使用，您可以使用 Rocket Pool 质押网站提取代表节点质押的 ETH。
:::

输入地址后，单击"查找"将带您进入下一个菜单，您可以在其中提取代表该节点质押的 ETH。

<img src="./images/withdraw_eth_on_behalf.png" width="100%" height="auto" />
