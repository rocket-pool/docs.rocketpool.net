# 从 Minipool 迁移到 Megapool 验证者

如果您是希望将 minipool 质押迁移到 megapool 的回归节点运营者，本页将提供分步指南说明如何操作。迁移包括退出您的 minipool、创建新的 megapool 验证者、取消质押旧版 RPL，然后在您的 megapool 上重新质押 RPL。megapool 验证者的质押金额为 4 ETH，这意味着您每拥有一个 8 ETH 的 minipool 验证者，就可以创建两个 megapool 验证者。


协议 ETH 收益的 9% 以[投票份额奖励](/zh/node-staking/megapools/staking-and-claiming-rewards#how-voter-share-is-distributed-to-megapool-rpl-stakers)的形式按比例分配给拥有 megapool 验证者的 RPL 质押者。
这是在之前标准 RPL 通胀奖励基础上的额外收益。



整个过程分为几个独立步骤：

1. 退出您的 minipool。即完成 minipool 并从信标链提取其全部余额。
2. 创建新的 megapool 验证者
3. [可选] 取消质押并提取旧版 RPL
4. [可选] 在 megapool 上重新质押 RPL。这样做将在 RPL 通胀奖励之外额外获得来自 ETH 收益的[投票份额奖励](/zh/node-staking/megapools/staking-and-claiming-rewards#how-voter-share-is-distributed-to-megapool-rpl-stakers)。

在决定将 minipool 质押迁移到 megapool 之前，您需要考虑 Rocket Pool 存款队列和信标链队列的状态。两者的等待时间都是可变的，且相互独立。Rocket Pool 存款队列管理新 megapool 验证者的创建以及它们何时与来自 rETH 质押者的 ETH 匹配。与此同时，信标链队列决定您的 minipool 验证者何时可以退出，更重要的是，它还决定您的新 megapool 验证者何时可以开始参与验证职责。

::: tip 实用链接
[saturn-1.net/queue](https://saturn-1.net/queue) 是由 Steely 创建的社区支持仪表板！此页面可以让您了解 Rocket Pool 存款队列的长度。

[validatorqueue.com](https://www.validatorqueue.com/) 是一个用于查看信标链队列长度的实用网站。该队列取决于进出信标链的 ETH 数量，以 `256 ETH 每轮次` 的速率处理。

[rocketpool.net/node-staking/rpl-fee-switch](https://rocketpool.net/node-staking/rpl-fee-switch) 可以模拟 Saturn 1 费用切换对您质押奖励的影响。

:::

::: warning 警告
关于获取奖励，有两点需要注意：

A. 除非您至少有一个活跃的 minipool 或 megapool 验证者，否则您的节点不会获得 RPL 通胀奖励。

B. 一旦您的 minipool 完全退出信标链，在您的 megapool 验证者开始主动验证之前，您将不会获得 ETH 奖励（佣金或投票份额）。
它需要经过 Rocket Pool 存款队列*和*信标链存款队列的处理，才能开始其验证职责。
您需要监控队列长度来决定何时迁移。

:::

::: warning 警告
Megapool 验证者需要经历信标链队列两次。
1. 预质押（1 ETH）。当来自存款池的 ETH 分配给您的 megapool 验证者时，只有 1 ETH 被发送到信标链。此时，您的验证者进入信标链队列。
2. 质押（31 ETH）。在验证者被信标链处理后，Rocket Pool 协议将验证该验证者的提款凭证，然后在信标链上质押剩余的 31 ETH。此时，验证者第二次也是最后一次进入信标链队列。

以前使用 minipool 时，[Oracle DAO（oDAO）](/zh/odao/overview#the-rocket-pool-oracle-dao)负责验证每个 minipool 验证者的提款凭证。这个新系统延长了队列时间，但消除了 oDAO 执行的信任要求。

[validatorqueue.com](https://www.validatorqueue.com/) 是监控信标链队列长度的实用工具。在将您的 minipool 质押迁移到 megapool 时，请考虑这一因素。

:::

## 第一步：退出您的 Minipool

退出您的 minipool 是将质押迁移到 megapool 的第一步。请点击以下链接，获取有关如何退出 minipool 并从信标链提取资金的详细指南：

https://docs.rocketpool.net/node-staking/withdraw

此过程通常需要几天时间，具体取决于信标链退出队列。您可以收藏此页面，在完成后再回来查看。

别忘了通过[分配全额奖励](/zh/node-staking/withdraw#distributing-full-rewards)来完成您的 minipool。
这将领取在 minipool 上获得的任何奖励，并允许您取消质押全额旧版 RPL。


## 第二步：创建 Megapool 验证者

一旦您的 minipool 关闭且 ETH 资金完全可访问，您就可以创建 megapool 验证者了！

以下页面将引导您完成创建新 megapool 验证者的所有步骤：

https://docs.rocketpool.net/node-staking/megapools/create-megapool-validator

作为回归节点运营者，每质押 4 ETH 的 minipool，您将获得一张快速队列票据。即每个 8 ETH 的 minipool 获得 2 张票据。
这些票据将自动提供给您。

Rocket Pool 存款队列以每 1 个标准队列验证者处理 4 个快速队列验证者的速率处理存款。
请阅读 [Rocket Pool 存款队列和快速队列](/zh/node-staking/megapools/create-megapool-validator#rocket-pool-deposit-queue-and-express-queue)，了解更多关于如何将 ETH 分配给新 megapool 验证者的信息。

## 第三步：取消质押并提取旧版 RPL

旧版 RPL 的最低质押要求是 minipool 借入 ETH 的 15%。如果您希望取消质押全额旧版 RPL，您需要先关闭所有 minipool（使借入 ETH 降至 0）。

准备好后，您可以使用以下命令取消质押并提取旧版 RPL：
```
rocketpool node withdraw-rpl
```
提取旧版 RPL 前有 28 天的取消质押等待期。
请点击以下链接获取有关如何取消质押和提取旧版 RPL 的详细说明。

https://docs.rocketpool.net/node-staking/megapools/staking-and-claiming-rewards#unstaking-and-withdrawing-rpl

## 第四步：在您的 Megapool 上质押 RPL

点击以下链接获取有关如何在 megapool 上质押 RPL 的说明：

https://docs.rocketpool.net/node-staking/megapools/staking-and-claiming-rewards#staking

此步骤完全可选，因为创建 megapool 验证者不需要质押 RPL。但如果您想获得投票份额奖励、RPL 通胀奖励（与之前的 RPL 通胀奖励相同），以及在[协议 DAO（pDAO）治理](/zh/pdao/overview)中拥有投票权，则不要错过此步骤。

- 投票份额（协议收益的 9%）根据各节点运营者在 megapool 总 RPL 质押中的份额按比例分配。

有关投票份额如何分配给拥有 megapool 验证者的 RPL 质押者的更详细信息，请阅读本指南：
[投票份额如何分配给 Megapool RPL 质押者](/zh/node-staking/megapools/staking-and-claiming-rewards#how-voter-share-is-distributed-to-megapool-rpl-stakers)

就这样完成了！如有需要，别忘了查阅[监控和维护](/zh/node-staking/maintenance/overview)指南。本节涵盖了保持节点良好运行的所有相关主题，包括软件更新和如何监控节点健康状况。
