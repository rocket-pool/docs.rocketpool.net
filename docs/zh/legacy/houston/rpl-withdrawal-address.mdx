# 设置您的 RPL 提款地址

重要的是要理解[主要提款地址](../../node-staking/prepare-node#setting-your-primary-withdrawal-address)和 RPL 提款地址之间的区别。默认情况下,您的 RPL 提款地址未设置,您的主要提款地址是您所有 RPL 检查点奖励、质押的 RPL 和信标链 ETH 将被发送到的地方。

如果您愿意,可以为 RPL [设置单独的提款地址](https://rpips.rocketpool.net/RPIPs/RPIP-31)。如果设置了 RPL 提款地址,它将能够触发并从通胀奖励中领取 RPL,并且不会影响 ETH 共识奖励或与 ETH 相关的任何内容。在这种情况下,您的主要提款地址将不再对 RPL 相关操作拥有任何权限。

这创造了一些有趣的机会,其中 RPL 可以由不希望接触 RPL 的节点运营者的实体提供给该运营者。然后,该实体可以领取 RPL 奖励,以提供节点所需的保险抵押品。换句话说,这允许不同的参与者提供每种抵押品类型。

::: warning 注意
首次设置节点时,您的 RPL 提款地址默认为未设置。
如果您选择设置 RPL 提款地址,**至关重要的是将其设置为由冷钱包控制的不同地址。**
冷钱包可以是硬件钱包(例如 [Ledger](https://www.ledger.com/)、[Trezor](https://trezor.io/) 或 [Grid+](https://gridplus.io/))或智能合约钱包(例如 [Safe](https://app.safe.global/welcome))。

我们**强烈建议**您不要使用 MetaMask 等热钱包作为您的 RPL 提款地址。

这样,如果您的节点钱包被入侵,攻击者无法通过强制您退出来访问您质押的 RPL,因为所有这些资金都将发送到您单独的冷钱包(他们希望无法访问)。

RPL 提款地址在节点运营者级别设置。
如果您创建多个迷你池,它们都将引用相同的 RPL 提款地址,因此您只需执行一次此设置。

[请参阅此处](https://ethereum.org/en/security/#wallet-security)以获取以太坊基金会关于钱包安全的一些基础知识。
:::

有两种不同的方法可以做到这一点。
请阅读下面的两个选项以确定哪一个适用于您。

完成此操作后,您将**无法再使用 `set-rpl-withdrawal-address` 命令更改 RPL 提款地址**。
要更改它,您需要从**活跃的** RPL 提款地址(您刚刚切换到的地址)发送签名交易。
Rocket Pool 网站有一个功能可以帮助您执行此操作。

如果您想取消设置 RPL 提款地址,过程非常简单。首先,您需要导航回 Rocket Pool 网站上的 RPL 提款地址页面。

如果您使用节点的 RPL 提款地址登录,您应该能够单击此处右侧的**取消设置**按钮:

单击**取消设置**将带您到一个页面,您可以在其中确认详细信息。如果您对看到的内容满意,请单击橙色**取消设置**按钮将交易发送到网络。
