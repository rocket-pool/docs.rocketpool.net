# 代表节点质押 ETH

RPIP-32 允许账户[代表](https://rpips.rocketpool.net/RPIPs/RPIP-32)已在协议中注册的 Rocket Pool 节点质押 ETH。这支持多种情况,其中节点运营者并非直接提供 ETH:

- 为节点运营者增强安全性,因为他们可以直接从硬件钱包质押,无需事先将资金转移到节点。
- 质押即服务提供商,其中资金托管由受信任的托管方管理。
- 协议集成,其中资金托管由智能合约管理。
- DAO 或组织,其中资金托管由金库管理。

虽然此功能的主要目的是促进单一存款人场景,但值得注意的是,多个独立存款人也可以通过在上层创建智能合约来利用此功能。Rocket Pool 还在我们之前的 Atlas 版本中引入了[代表节点质押 RPL](../../node-staking/create-validator#whitelisting-an-address-to-stake-on-behalf) 的能力。

### 前提条件

确保您已正确配置了您打算资助的节点的[主提款地址](../../node-staking/prepare-node#setting-your-primary-withdrawal-address)。请点击上面的链接查看 houston 后提款地址如何运作的指南。

::: tip 注意
不要跳过此步骤!所有 ETH 保证金/奖励将被提取到主提款地址。换句话说,为节点提供 ETH 的地址应该用作主提款地址!请确保节点的主提款地址设置为您拥有托管权的钱包。
:::
