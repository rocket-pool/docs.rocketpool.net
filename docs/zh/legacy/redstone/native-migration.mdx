# [原生模式] Redstone 更新和合并指南

本指南将涵盖在使用**原生模式**时,为 Redstone 更新和合并准备节点所需了解的所有内容。

## 升级到 v1.5.0 之前需要做的事情

在升级到 Smartnode v1.5.0 及更高版本之前,请检查以下清单以确保您已准备就绪:

### 切换到完整执行客户端

合并要求您运行自己的执行客户端,因此您将无法再使用 Infura 或 Pocket 等远程提供商。

由于这一变化,如果您当前使用的是轻量级执行客户端,您应该在仍使用 v1.4 时切换到完整客户端,让它同步完成,然后再升级到 v1.5。

### 设置引擎 API

合并改变了执行客户端与共识客户端的通信方式。
合并不再使用基于旧的 HTTP 或 Websocket 的 RPC 系统,而是需要执行客户端公开的新系统,称为引擎 API。

这是一个特殊连接,让共识客户端用权益证明替代旧的工作量证明挖矿系统;这是合并的核心。
它还通过密钥令牌进行**身份验证**,因此只有您的共识客户端可以连接到您的执行客户端 - 其他任何东西都不行。

由于您管理自己的执行客户端和共识客户端,您需要手动设置引擎 API。
如何操作完全取决于您运行的客户端。

CoinCashew 有[一份很棒且简明的指南](https://www.coincashew.com/coins/overview-eth/ethereum-merge-upgrade-checklist-for-home-stakers-and-validators),介绍如何在执行客户端和共识客户端上设置引擎 API。
看看那个指南,并在升级之前通过确保它仍然正常证明来测试新配置。

我们将在下面向您展示如何设置验证器客户端,使其自动使用 Smartnode 软件所需的正确费用接收者。

## 升级到 v1.5.0

将 Smartnode 栈升级到 v1.5.0 与任何其他升级没有什么不同。
只需遵循[此处的常规说明](../../node-staking/updates#updating-the-smartnode-stack)。

## 升级后应该做的事情

在原生模式下,升级后您需要手动完成几件事:

### 确保升级成功

首先要做的是确保您的节点正常工作。
考虑采取以下步骤:

- 检查执行客户端、共识客户端、验证器客户端和 Smartnode 守护程序(`rp-node` 服务)的日志脚本,以确保它们都正常运行且没有错误。
- 通过区块浏览器(例如您的 Grafana 仪表板和 [https://beaconcha.in](https://beaconcha.in))确认您仍在正确证明
  - 请记住,如果您启用了 Doppelganger 保护,重启后您会错过几次证明。这是正常的!

### 在验证器客户端中设置费用接收者

在合并之前需要设置的**关键**细节之一是验证器客户端指定的**费用接收者**。
如[概述文章](./whats-new#fee-recipients-and-your-distributor)中所述,这可以是以下两个值之一:

- 如果您**选择加入**平滑池,这必须是**平滑池合约**的地址。您可以从[官方合约页面](/zh/protocol/contracts-integrations)获取地址。
- 如果您**未**加入平滑池,这必须是您节点的**费用分配器合约**的地址。您可以通过运行 `rocketpool node status`,在 `Fee Distributor and Smoothing Pool` 部分下获取地址。

在原生模式下,如果您使用 Smartnode 守护程序服务 `rp-node`,您可以选择让 Smartnode 为您管理此项,或者如果您不使用守护程序,则自行管理。

#### 通过守护程序自动管理

Smartnode 守护程序将自动确定您节点的适当费用接收者,并在其发生变化时进行管理(例如选择加入和退出平滑池)。
这是**最安全的**选项,因为 Smartnode 将始终确保将其设置为**防止[处罚](https://github.com/rocket-pool/rocketpool-research/blob/master/Penalties/penalty-system)**的值。

它的工作方式是维护一个包含正确费用接收者的文件,并定期刷新以确保其正确性。
当需要更新时,它会修改文件并自动重启您的验证器客户端,以便加载新接收者 - 类似于在质押新 minipool 后重启验证器客户端的方式。

选择您的客户端以了解如何设置:

#### 手动费用接收者管理

::: danger 警告
通过这样做,您将承担全部责任,确保您的费用接收者始终设置为正确的地址。

请阅读[处罚规范](https://github.com/rocket-pool/rocketpool-research/blob/master/Penalties/penalty-system)以了解根据您的配置必须将其设置为什么,以及何时可以安全地从一个值更改为另一个值。

如果不这样做,可能会导致您的 minipool 受到处罚!
:::

**在 Redstone 部署之前**,您可以简单地使用您所在网络的 rETH 地址(可以在[官方合约页面](/zh/protocol/contracts-integrations)中找到)。
无论如何,rETH 地址始终是安全的。

**一旦部署了 Redstone**,您可以通过 `rocketpool node status` 查看应该将费用接收者设置为的确切地址。例如,如果您选择加入平滑池,它将显示平滑池的地址,并注明您必须将其用作费用接收者:

<img src="../../node-staking/images/native-fee-rc-in.png" width="100%" height="auto" />

如果您_未_选择加入平滑池,它将显示您的费用分配器地址,并注明您必须将其用作费用接收者:

<img src="../../node-staking/images/native-fee-rc-out.png" width="100%" height="auto" />

选择您的共识客户端以了解如何配置。

### 设置 MEV-Boost

[MEV-boost](https://boost.flashbots.net/) 是 Flashbots 提供的系统,在合并后为权益证明验证器提供 MEV 奖励。

**Rocket Pool 要求所有节点使用它以最大化回报,从而保持协议与其他质押服务的竞争力。**

您需要对信标节点/共识客户端进行一些调整,以将其连接到 MEV-boost。

MEV-boost 目前在 Hoodi 或主网上不可用,因此您此时不需要设置它。
当然,在此过渡期间,您不会因未使用它而受到处罚。

一旦它可用,我们将宣布一个日期,届时必须安装并连接到您的节点。
Flashbots 届时将提供您可以遵循的说明,我们将在此处链接到它们。

::: danger 注意
一旦我们宣布所有节点运营商必须启用 MEV-boost,您必须确保已正确安装并配置了信标节点!

不这样做将导致您的 minipool 受到[处罚](https://github.com/rocket-pool/rocketpool-research/blob/master/Penalties/penalty-system)。
:::

### 设置后备节点

由于合并与 Infura 和 Pocket 等远程提供商不兼容,当您的主执行客户端离线时,您将失去将它们用作后备执行客户端的能力。

Smartnode 仍然具有提供后备执行客户端(现在也提供后备共识客户端)的能力,但您现在需要使用您控制的执行客户端和共识客户端。

有关设置后备节点的更多信息,请参阅[后备节点指南](../../node-staking/fallback)。

### 初始化您的费用分配器

如果您不打算选择加入[平滑池](./whats-new#smoothing-pool)并将所有优先费用和 MEV 奖励索取到您的[费用分配器合约](./whats-new#fee-recipients-and-your-distributor),您最终将必须初始化它(在链上创建合约实例)才能将奖励索取到您的提款地址。

这是一个相当便宜的操作,只需要执行一次。

::: tip 提示
初始化您的费用分配器可以在**任何时候**完成。
您可以在初始化之前很久就让奖励在其地址中累积,并且您的余额在初始化后将保留。

我们建议您在 gas 价格较低时这样做,以尽量减少开销成本。

请注意,**必须初始化才能索取您的奖励。**
:::

### 选择加入平滑池

如果您打算立即利用[平滑池](./whats-new#smoothing-pool),您应该在第一个 Redstone 奖励周期结束之前选择加入,以最大化您的"资格"金额。

可以通过运行以下命令来选择加入:

```shell
rocketpool node join-smoothing-pool
```

### 索取奖励

Redstone 升级用[全新系统](./whats-new#new-rewards-system)替换了昂贵且有问题的旧奖励系统,该系统更便宜,支持 RPL 的自动重新质押(部分和全部金额),并且 - 最重要的是 - **让您可以随时索取奖励**。

由于不再有索取奖励的时间限制,并且一次索取多个奖励间隔更便宜,Smartnode 的自动奖励索取功能**已被删除**。
您现在可以通过以下命令索取奖励:

```shell
rocketpool node claim-rewards
```

这将向您显示从 Redstone 升级开始的所有奖励间隔中累积的所有奖励。
