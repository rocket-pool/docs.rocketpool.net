# 从以前的 Beta 测试迁移智能节点

如果您在我们之前的 Medalla 或 Pyrmont 的 beta 测试期间是 Rocket Pool 节点运营者,感谢您帮助我们改进协议,欢迎回来!
这个简短的指南将引导您完成清除自己的 beta 版本并为 Hoodi 测试网或主网上的真实质押设置最新版本的过程。

<div className="p-3">
  <Tabs>
    <Tab label="Hoodi Network">
      <p className="rspress-directive warning">
        <p className="rspress-directive-title">注意</p>
        您需要创建一个新的节点钱包来与 Hoodi 测试网络交互。
        重复使用以前 beta 版本的钱包可能会导致创建新验证者时失败!
      </p>

      从 beta 版本升级到 Hoodi 测试网络有三个主要步骤。

      首先,将您拥有的任何 hoodi ETH 迁移到单独的钱包。
      然后,您可以在重新安装 Rocket Pool 后创建新的节点钱包后将其迁移回去,这样您就不需要太依赖 hoodi ETH 的水龙头。

      最简单的方法是通过 Metamask - 请参阅 [rETH 质押指南](../liquid-staking/overview)以了解如何设置新钱包的演练。
      您可以使用以下命令执行此操作:

      ```shell
      rocketpool node send XXX eth 0xabcd1234...
      ```

      其中 `XXX` 是要发送的 hoodi ETH 数量,`0xabcd1234...` 是您创建的新钱包的地址。
      **请注意,您需要留下一点 ETH 来支付交易中使用的 gas 费!**

      接下来,使用以下命令删除所有旧的 Docker 镜像和链数据:

      ```shell
      rocketpool service terminate
      ```

      完成后,删除旧的 Rocket Pool 数据和设置文件夹。
      **这将删除您的旧钱包和旧验证者,所以请确保您不再需要保存它们!**

      ```shell
      sudo rm -rf ~/.rocketpool
      ```

      完成后,您的旧 beta 设置将被清理。
      您可以按照 [Rocket Pool 安装指南](../node-staking/eth-clients)来了解如何为 Hoodi 测试网安装和配置 Rocket Pool。
