# Migration du Smartnode depuis les Tests Bêta Précédents

Si vous avez été un opérateur de nœud Rocket Pool durant nos précédents tests bêta sur Medalla ou Pyrmont, merci de nous avoir aidé à améliorer le protocole et bienvenue de retour !
Ce bref guide vous guidera à travers le processus de purge de votre ancienne build bêta et de configuration de la nouvelle build soit pour le testnet Hoodi, soit pour le staking réel sur mainnet.

<div className="p-3">
  <Tabs>
    <Tab label="Réseau Hoodi">
      <p className="rspress-directive warning">
        <p className="rspress-directive-title">REMARQUE</p>
        Vous devrez créer un nouveau portefeuille de nœud pour interagir avec le réseau de test Hoodi.
        La réutilisation d'un portefeuille d'une build bêta précédente peut entraîner des échecs lors de la création de nouveaux validateurs !
      </p>

      Il y a trois étapes majeures pour passer d'une build bêta au réseau de test Hoodi.

      Tout d'abord, migrez tout ETH hoodi que vous possédez vers un portefeuille séparé.
      Vous pouvez ensuite le migrer vers le nouveau portefeuille de nœud que vous créerez après la réinstallation de Rocket Pool, afin de ne pas avoir à dépendre autant d'un robinet pour l'ETH hoodi.

      La méthode la plus simple pour ce faire est via Metamask - consultez le [guide de staking rETH](../liquid-staking/overview) pour une procédure pas à pas sur la création d'un nouveau portefeuille.
      Vous pouvez le faire en utilisant la commande suivante :

      ```shell
      rocketpool node send XXX eth 0xabcd1234...
      ```

      où `XXX` est le montant d'ETH hoodi à envoyer, et `0xabcd1234...` est l'adresse du nouveau portefeuille que vous avez créé.
      **Notez que vous devrez laisser un peu d'ETH pour payer le gaz utilisé dans la transaction !**

      Ensuite, supprimez toutes les anciennes images Docker et données de chaîne avec la commande suivante :

      ```shell
      rocketpool service terminate
      ```

      Une fois cela fait, supprimez l'ancien dossier de données et paramètres Rocket Pool.
      **Cela supprimera votre ancien portefeuille et vos anciens validateurs, assurez-vous donc de ne plus avoir besoin de les sauvegarder pour quoi que ce soit !**

      ```shell
      sudo rm -rf ~/.rocketpool
      ```

      Une fois cela fait, votre ancienne configuration bêta sera nettoyée.
      Vous pouvez suivre le [guide d'installation de Rocket Pool](../node-staking/eth-clients) pour une procédure pas à pas sur l'installation et la configuration de Rocket Pool pour le testnet Hoodi.
