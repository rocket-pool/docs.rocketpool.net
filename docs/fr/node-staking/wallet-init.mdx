# Créer un nouveau portefeuille

Si les services Smartnode sont opérationnels, l'étape suivante consiste à configurer un portefeuille Ethereum pour votre nœud.
Il s'agit simplement d'une adresse de la couche d'exécution qui contiendra les fonds de votre nœud - elle sera utilisée pour envoyer de l'ETH à votre minipool lorsque vous commencerez le staking, payer les frais de gas lors de diverses transactions et autres opérations diverses.

Vous avez la possibilité soit de créer un tout nouveau portefeuille à partir de zéro, soit d'importer ou de récupérer un portefeuille ou une adresse existante.
Veuillez choisir l'option souhaitée parmi les onglets ci-dessous.

<div className="p-3">
  <Tabs>
    <Tab label="Créer un nouveau portefeuille">
      La façon la plus courante d'exécuter un nœud est de créer une nouvelle adresse Ethereum dédiée au nœud.
      L'interface CLI Smartnode offre un moyen simple de le faire :

      ```shell
      rocketpool wallet init
      ```

      Vous serez d'abord invité à saisir un mot de passe pour protéger la clé privée de votre portefeuille.
      Ensuite, vous recevrez la **phrase mnémonique unique de 24 mots** pour votre nouveau portefeuille.
      Il s'agit de la **phrase de récupération** de votre portefeuille.
      Si vous perdez votre machine, vous pouvez utiliser cette phrase pour régénérer votre portefeuille et restaurer tous les validateurs de la couche de consensus qui y sont attachés.

      <p className="rspress-directive warning">
        <p className="rspress-directive-title">REMARQUE</p>
        Il est **essentiel** que vous notiez cette phrase mnémonique car c'est la seule fois qu'elle vous sera montrée, mais conservez-la dans un endroit sûr.
        Toute personne disposant de cette phrase peut prendre le contrôle de votre portefeuille.
      </p>

      Une fois que vous avez confirmé votre phrase mnémonique, vous recevrez l'adresse Ethereum unique de votre portefeuille de nœud.
    </Tab>
    <Tab label="Récupérer ou importer un portefeuille existant">
      Une fois que vous avez terminé de saisir la phrase mnémonique, vous verrez un message de succès comme celui-ci :
      "Le portefeuille du nœud a été initialisé avec succès.
        Compte du nœud : LongueChaineDeLettresEtDeChiffres"
      Il s'agit de l'adresse publique à laquelle vous enverrez de l'ETH dans "Préparer votre nœud pour le fonctionnement". Vous pouvez également la vérifier en tapant "rocketpool wallet status".

      Si vous avez déjà un portefeuille de nœud que vous avez créé lors d'une installation précédente du Smartnode, ou si vous souhaitez importer une adresse que vous avez créée ailleurs (comme avec **Allnodes**), veuillez suivre les instructions du guide [Importer un portefeuille existant pour votre nœud](./recovering-rp).

      Revenez ici une fois que vous avez terminé.
    </Tab>

  </Tabs>
</div>

::: warning REMARQUE
Depuis la version actuelle, Rocket Pool doit avoir accès à la clé privée de votre portefeuille afin d'effectuer ses tâches automatiques.
**Cela signifie que la clé privée existera dans un fichier sur votre machine.**
Si un attaquant parvient à accéder à votre machine, il pourrait accéder à votre portefeuille de nœud et voler tous les tokens qu'il contient !
Veuillez vous assurer que vous avez suivi le guide de sécurité dans la section [Sécuriser votre nœud](./securing-your-node) avant de continuer, et que vous êtes à l'aise avec votre posture de sécurité.
:::

    <Tab label="Modes Docker et Hybride">
      La clé privée de votre portefeuille sera stockée dans un fichier situé à `~/.rocketpool/data/wallet`.

      Le mot de passe utilisé pour chiffrer la clé de votre portefeuille sera stocké dans `~/.rocketpool/data/password`.
      C'est le fichier qui doit être protégé en permanence.
    </Tab>
    <Tab label="Mode natif">
      Les fichiers de portefeuille et de mot de passe seront stockés dans le répertoire `data` sous le répertoire Rocket Pool que vous avez configuré précédemment (par exemple `/srv/rocketpool`).

      La clé privée de votre portefeuille sera stockée dans un fichier situé à `/srv/rocketpool/data/wallet`.

      Le mot de passe utilisé pour chiffrer la clé de votre portefeuille sera stocké dans `/srv/rocketpool/data/password`.
      C'est le fichier qui doit être protégé en permanence.
    </Tab>
