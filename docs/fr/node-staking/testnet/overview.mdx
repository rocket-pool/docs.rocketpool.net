# Pratique avec le Réseau de Test

Un **réseau de test** Ethereum (`testnet` en abrégé) est une blockchain Ethereum, fonctionnellement identique à la blockchain Ethereum standard, mais elle est destinée à être utilisée pour tester des contrats intelligents avant qu'ils ne soient mis en service sur la blockchain "réelle" principale (appelée `mainnet`).
Les réseaux de test fonctionnent en parallèle avec le réseau principal - il y en a en fait plusieurs en cours d'exécution en ce moment !

Vous pouvez en savoir plus sur certains des réseaux de test de la couche d'exécution [ici](https://support.mycrypto.com/how-to/getting-started/where-to-get-testnet-ether) si vous le souhaitez.
Rocket Pool utilise **Hoodi** pour son réseau de test de couche d'exécution.
Vous pouvez utiliser un explorateur de blocs pour celui-ci ici si vous souhaitez y jeter un œil : [https://hoodi.etherscan.io/](https://hoodi.etherscan.io/)

Le même concept s'applique à la couche de consensus également.
Rocket Pool utilise **Hoodi** pour son réseau de test de couche de consensus.
Vous pouvez trouver un explorateur de blocs pour celui-ci ici : [https://hoodi.beaconcha.in/](https://hoodi.beaconcha.in/)

Parce que les réseaux sont destinés aux tests, **l'ETH sur ceux-ci est distribué gratuitement pour aider les gens à tester**.
Cet ETH **ne peut pas être transféré** vers le réseau principal ; il doit rester sur Hoodi et n'a pas de valeur réelle, vous pouvez donc le considérer comme **de l'argent Monopoly**.

::: danger
**N'utilisez PAS votre vrai ETH du réseau principal si vous essayez le réseau de test de Rocket Pool !**
Vous **perdrez vos fonds** pour toujours si vous faites cela !
Vous ne devriez utiliser que l'ETH de test que vous avez reçu sur le réseau Hoodi à partir d'un robinet (décrit ci-dessous) !
:::

## Configuration du Réseau de Test

[La configuration par défaut de Rocket Pool utilisant Docker](../install-modes) gère automatiquement tous les paramètres impliqués dans l'utilisation d'un réseau de test pour vous.
Il n'y a rien de spécial que vous devez faire.

Si vous utilisez [une configuration différente](../install-modes) avec un client d'exécution ou de consensus externe, ou si vous exécutez vos clients [nativement sans Docker](../install-modes), vous devrez configurer cela manuellement :

- Pour **Geth** : ajoutez `--hoodi` à la commande de lancement.
- Pour **Lighthouse** : ajoutez `--network hoodi` à la commande de lancement.
- Pour **Nimbus** : ajoutez `--network=hoodi` à la commande de lancement.
- Pour **Prysm** : ajoutez `--hoodi` à la commande de lancement.
- Pour **Teku** : ajoutez `--network=hoodi` à la commande de lancement.
- Pour **Lodestar** : préfixez la commande de lancement avec `LODESTAR_NETWORK=hoodi`

Consultez la section suivante pour apprendre comment obtenir de l'ETH de test sur Hoodi, dont vous aurez besoin pour tester Rocket Pool.

## Obtenir de l'ETH de Test sur Hoodi

Pour utiliser le réseau de test Rocket Pool, vous aurez besoin d'ETH sur Hoodi.
Heureusement, vous pouvez obtenir cela gratuitement en utilisant un **robinet** - un service automatisé qui vous fournira de l'ETH Hoodi pour tester.

::: warning NOTE
Vous devrez fournir une adresse Ethereum pour recevoir de l'ETH de test.

Si vous essayez simplement de staker votre ETH normalement et de recevoir du rETH, veuillez consulter le [Guide de Staking](../../liquid-staking/overview) avant d'obtenir de l'ETH Hoodi.
Vous serez invité à revenir ici à l'étape appropriée.

Si vous prévoyez d'exploiter un nœud, veuillez parcourir le [Guide de l'Opérateur de Nœud](../responsibilities) pour configurer un nœud et créer d'abord un nouveau portefeuille.
Vous serez invité à revenir ici à l'étape appropriée.
:::

Rocket Pool fournit un robinet sur son [serveur Discord](https://discord.gg/rocketpool).
Rejoignez le serveur et dirigez-vous vers le canal `#support`.

Ici, vous pouvez demander de l'ETH de test.
Vous devrez fournir l'adresse de votre nœud.

::: warning NOTE
Le bot de robinet du réseau de test n'est plus directement accessible au public en raison d'abus antérieurs.
Il n'est disponible que pour l'équipe de développement et les membres éminents de la communauté.
Pour recevoir de l'eth de test, exécutez les commandes ci-dessous, indiquez vos intentions dans le canal.
Soyez patient et quelqu'un accédera au robinet en votre nom.
:::

Si vous ne prévoyez pas d'enregistrer un nœud et que vous voulez simplement de l'ETH pour tester le staking afin de recevoir du rETH, tapez la commande suivante dans le canal :

```
!hoodiEth <votre adresse hoodi>
```

sans les crochets.
Si votre portefeuille n'a pas déjà d'ETH, le bot enverra à votre portefeuille 1 ETH pour tester.

Si vous _prévoyez_ d'exploiter un nœud, procédez comme suit :

1. Commencez par exécuter la commande `!hoodiEth` ci-dessus pour acquérir suffisamment d'ETH pour enregistrer votre nœud
2. Enregistrez votre nœud (consultez la page [Préparation de votre Nœud](../prepare-node) pour les instructions)
3. Exécutez la commande suivante pour obtenir 8 ETH Hoodi afin de pouvoir créer un validateur :

```
!hoodiOperator <votre adresse hoodi>
```

Pour utiliser `!hoodiOperator`, l'adresse que vous fournissez _doit_ être un nœud Rocket Pool enregistré.
Par conséquent, vous devez suivre ces 3 étapes dans cet ordre.

## Obtenir du RPL de Test sur Hoodi

Si vous testez l'exploitation de nœud, vous aurez besoin de RPL de test en plus de l'ETH de test.

Rocket Pool fournit un robinet sur son [serveur Discord](https://discord.gg/rocketpool).
Rejoignez le serveur et dirigez-vous vers le canal `#support`.

Ici, vous pouvez demander du RPL de test.
Vous devrez fournir l'adresse de votre nœud.

::: warning NOTE
Le bot de robinet du réseau de test n'est plus directement accessible au public en raison d'abus antérieurs.
Il n'est disponible que pour l'équipe de développement et les membres éminents de la communauté.
Pour recevoir du RPL de test, exécutez les commandes ci-dessous, indiquez vos intentions dans le canal.
Soyez patient et quelqu'un accédera au robinet en votre nom.
:::

```
!hoodiRpl <votre adresse de nœud hoodi>
```

Cela vous fournira 600 du token **RPL hérité (v1)**, qui est analogue au token RPL original sur le réseau principal.
Rocket Pool utilise un **nouveau token RPL (v2)** qui prend en charge l'inflation utilisée pour récompenser les opérateurs de nœuds.

Pour apprendre à échanger du RPL hérité contre du nouveau RPL, suivez les étapes du guide [Création d'un Minipool (Validateur)](../create-validator).
